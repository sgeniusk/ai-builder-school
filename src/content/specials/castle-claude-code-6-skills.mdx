## 01 · `grill-me` — 받자마자 코드 대신 질문

코드를 쓰기 전, AI가 사용자에게 거꾸로 핵심 질문 3~10개를 던집니다. 답이 모이면 그제서야 의도가 명료해집니다.

## 03 · `to-issues` — 수평 X · 수직 ✓

PRD가 만들어진 직후 작업 단위로 쪼갭니다. **200줄 PR 1개보다 20줄 PR 10개가 안전.**

  left={{

      { text: "→ 다 합쳐야만 동작", emphasis: true },

  }}
  right={{
    heading: "수직 슬라이스 · GOOD",

    items: [
      { text: "Issue 1 — 비로그인 결제 차단" },
      { text: "Issue 2 — 카드 결제 1건" },
      { text: "Issue 3 — 결제 실패 메시지" },
      { text: "→ 각자 끝나면 그것만으로 동작", emphasis: true },

**효과** — PR 리뷰 시간 1/5 · 매일 production에 진척 · 실패해도 한 슬라이스만 되돌리면 끝.

## 04 · `caveman` — 같은 답, 4분의 1 길이

Julius Brussee 케이스 — fluff를 제거해 응답 토큰 75% 절감. "동굴인 말투"로 응답.

<TwoColumnListBlock
  left={{
    heading: "BEFORE · 일반 모드 ~280토큰",
    icon: "alert-triangle",
    items: [

      { text: "\"The issue is likely an off-by-one error in the auth middleware...\"" },
      { text: "fluff가 한 세션에서 수십 KB 누적", emphasis: true },
      { text: "컨텍스트도, 비용도, 속도도 손해" },
    ],
  }}
  right={{
    heading: "AFTER · caveman full ~60토큰",
    icon: "zap",
    items: [
      { text: "\"Bug in auth middleware.\"" },
      { text: "\"Token expiry check use `<` not `<=`. Fix:\"", emphasis: true },
      { text: "제거 — a/an/the, pleasantries", emphasis: true },
      { text: "유지 — 기술 용어·코드·에러 메시지 (정확성 100%)" },
      { text: "토글 — `/caveman lite | full | ultra`" },
    ],
  }}
/>

코드·커밋·PR은 정상 문체로 유지 — 캐주얼 모드는 꺼집니다.

## 05 · `handoff` — 긴 세션은 결국 잃어버린다

컨텍스트가 가득 차기 전, 현재 세션 상태를 `HANDOFF.md` 한 파일에 정리. 새 세션은 이 파일 읽고 시작.

## 06 · `improve-codebase` — Deep module 원칙

John Ousterhout, *A Philosophy of Software Design*. 좋은 모듈은 **바깥에 노출되는 면이 작고, 안에서 하는 일이 많다.**

<TwoColumnListBlock
  left={{
    heading: "Shallow module · BAD",
    icon: "alert-triangle",
    items: [
      { text: "많은 함수 노출" },
      { text: "각 함수는 단순 래퍼" },
      { text: "호출자가 내부를 다 알아야 함", emphasis: true },
      { text: "→ 추상화가 적자", emphasis: true },
    ],
  }}
  right={{
    heading: "Deep module · GOOD",
    icon: "check-circle",
    items: [
      { text: "한두 개 함수만 노출", emphasis: true },
      { text: "내부에서 복잡도를 흡수" },
      { text: "호출자는 인터페이스만 알면 됨" },
      { text: "→ 추상화가 흑자", emphasis: true },
    ],
  }}
/>

**사용 시점** — 큰 PR 끝났을 때 정기 점검 · 비슷한 코드가 3곳에서 보일 때 · 디렉터리 구조가 직관과 어긋날 때. 예시 명령 — `improve-codebase: shallow한 모듈·중복 패턴·죽은 코드 찾아서 리포트만. 수정은 다음 단계.`
