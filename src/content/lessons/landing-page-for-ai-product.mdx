## 만든 것이 보이지 않으면 존재하지 않는 것이다

MVP를 만들고 링크를 공유했다. 클릭이 없거나, 클릭했는데 이탈한다.

문제는 대부분 랜딩 페이지에 있다.

```
"AI로 회의 요약을 자동화합니다."
→ 그래서 나한테 왜 필요한데?
```

기능 설명은 제품을 소개하지 못한다. 통증이나 결과로 시작해야 처음 보는 사람이 멈춘다.

---

## 5단 구조

랜딩 페이지의 순서는 방문자의 마음이 움직이는 순서와 같다.

### 1. Hook — 3초 안에 "나 얘기"로 시작

기능 설명이 아니라 통증 또는 결과.

```
❌ "AI 회의 요약 서비스입니다."
✅ "회의가 끝나고 30분 동안 뭘 정리하고 있나요?"
✅ "회의 요약, AI가 5초 만에 해드립니다."
```

Hook 3가지 버전을 써보고 가장 반응이 좋은 걸 쓴다:
- 통증 버전: "아직도 회의 요약에 30분씩 쓰고 있나요?"
- 결과 버전: "회의 끝나면 5초 만에 요약본이 생깁니다."
- 질문 버전: "마지막 미팅에서 뭘 결정했는지 기억하시나요?"

### 2. Problem — 공감

방문자가 "맞아, 나도 그래"라고 느끼는 구간.

```
클라이언트 미팅 후 메모, 정리, 공유까지
매번 30분 이상이 사라집니다.
그리고 한 달 뒤에 "그때 뭐라고 했었지?" 하게 됩니다.
```

### 3. Solution — 제품이 하는 일

기능이 아니라 변화를 설명한다.

```
[제품명]은 회의 녹음을 올리면
결정 사항·액션 아이템·요약을 자동으로 정리합니다.
30분이 5초로 줄어듭니다.
```

### 4. Social Proof — 신뢰

초기에는 베타 사용자 한 줄 후기로 충분하다.

```
"회의 직후 팀원들한테 요약본 바로 보내줄 수 있어서
제일 좋았어요." — 프리랜서 디자이너 박지영
```

숫자가 있으면 더 좋다: "베타 사용자 23명이 주 3회 이상 사용 중"

### 5. CTA — 단 하나의 행동

버튼 하나, 링크 하나.

```
✅ "무료로 시작하기"
✅ "대기 목록 등록"
❌ "더 알아보기 / 시작하기 / 팀 플랜 보기" (세 개는 안 된다)
```

## AI로 카피 초안 생성하기

브리프를 AI에게 주고 요청한다:

```
이 AI 제품의 랜딩 페이지 카피를 써줘.
5단 구조로: Hook / Problem / Solution / Social Proof / CTA.

제품 브리프:
[브리프 내용]

각 섹션을 3-4줄 이내로 써줘.
```

생성된 카피를 그대로 쓰지 않는다. AI가 쓴 카피는 일반적이다.

**직접 편집해야 하는 두 곳:**
1. Hook — 내 페르소나의 언어로 바꾼다
2. Social Proof — 실제 베타 사용자 말로 교체한다

## 퍼블리시 방법

복잡하게 생각하지 않는다. 목적은 URL 하나를 만드는 것이다.

| 방법 | 시간 | 적합한 상황 |
|------|------|------------|
| Framer | 30분 | 비주얼이 중요할 때 |
| Notion 퍼블리시 | 10분 | 빠르게 확인할 때 |
| GitHub Pages | 20분 | 코드로 직접 만들고 싶을 때 |

---

## 검증 — 3명 테스트

퍼블리시 후 3명에게 링크를 보내고 묻는다:

```
"이 페이지 보고 10초 안에 뭘 하는 제품인지 알겠어?"
```

2명 이상이 이해하면 Hook이 작동한 것이다.
이해 못 하면 Hook을 바꾼다. 나머지 구조를 바꾸는 게 아니다.

---

## 핵심 원칙

> AI가 카피를 쓰고, 내가 내 페르소나의 언어로 바꾼다.
> 랜딩 페이지는 완성이 아니라 첫 번째 가설이다.
