# 장문 처리 전략 비교 노트

> 이 파일은 AI Builder School · Lesson 2.3 에서 제공되는 출발 템플릿입니다.
> 같은 문서, 같은 질문, 다른 전략 — 결과를 직접 측정해서 자기 도메인의 기본 전략을 결정하세요.

## 0. 실험 설계

| 항목 | 값 |

| 측정 모델 | |
| 측정 일자 | |

## 1. 전략 1 — Full Context (통째로)

### 실행 메모
- 문서를 PDF → 텍스트 변환 후 한 번에 시스템 프롬프트에 첨부
- 호출 1회

### 결과 답변

```
(여기에 모델 답을 그대로 붙여넣기)
```

### 측정

| 측정 | 값 |
| --- | --- |

| 출력 토큰 (대략) | |
| 호출 비용 (USD) | |
| 응답 시간 (초) | |
| 정확성 (1~5) | |
| 완전성 (앞·중간·끝 다 다뤘나, 1~5) | |

## 2. 전략 2 — Map-Reduce

### 실행 메모
- 문서를 5~7 청크로 분할
- 각 청크별 *"이 청크의 핵심 결정 1~3개"* 요약 (Map)
- 요약들을 다시 통합 프롬프트로 종합 (Reduce)

### 결과 답변

```
(여기에 최종 통합 답을 그대로 붙여넣기)
```

### 측정

| 측정 | 값 |
| --- | --- |
| 호출 수 | (예) 6 (5 map + 1 reduce) |
| 누적 입력 토큰 | |
| 누적 출력 토큰 | |
| 호출 비용 (USD) | |
| 총 응답 시간 (초) | |
| 정확성 (1~5) | |
| 완전성 (1~5) | |
| 청크 경계에서 끊긴 정보 있었나 | |

---

## 3. 전략 3 — Selective Injection

### 실행 메모
- 질문에서 키워드 추출 (예 — *"의사결정", "결정", "발표", "승인"*)
- 문서 내 키워드 매칭으로 상위 3 청크 선별
- 선별된 청크 + 질문만 모델에 주입

### 결과 답변

```
(여기에 모델 답을 그대로 붙여넣기)
```

### 측정

| 측정 | 값 |
| --- | --- |
| 선별된 청크 수 | |
| 입력 토큰 (대략) | |
| 출력 토큰 (대략) | |
| 호출 비용 (USD) | |
| 응답 시간 (초) | |
| 정확성 (1~5) | |
| 완전성 (1~5) | |
| 빠진 결정 있었나 | |

## 6. 다음 단계

- Selective 가 가장 좋았다면 — Phase 6 (RAG) 으로 가서 자동화
- Map-Reduce 가 가장 좋았다면 — Phase 3 의 자동화 파이프라인에 패턴으로 박기
- Full 이 충분했다면 — 문서 길이가 *임계점* 을 넘지 않는 한 그대로 유지
