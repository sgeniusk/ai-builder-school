## 터미널은 무섭지 않다

검은 화면에 커서만 깜빡인다. 뭘 쳐야 할지 모른다. 잘못 입력하면 컴퓨터가 망가질 것 같다.

전부 오해다.

터미널은 컴퓨터에게 텍스트로 명령하는 창일 뿐이다. 잘못 입력해도 `Ctrl+C`로 멈추거나 `cd ~`로 홈으로 돌아오면 된다. **파일을 실수로 삭제하지 않는 한 터미널 명령어로 컴퓨터가 망가지지 않는다.**

<Terminal title="처음 열었을 때" lines={[
  "Last login: Wed May 22 10:24:18 on ttys000",
  "$",
]} />

`$`는 명령을 받을 준비가 됐다는 신호. 그 뿐이다.

## 터미널 열기

- **Mac** — Spotlight(`Cmd+Space`) → "Terminal" 검색 → Enter. 끝.
- **Windows** — [git-scm.com](https://git-scm.com)에서 Git 설치 → 시작 메뉴 "Git Bash" 실행. **Git Bash 권장** (이 학교 명령어가 Mac과 똑같이 동작). PowerShell·WSL은 명령어가 다르거나 설정 부담이 큼.

이후 모든 명령어는 Mac/Git Bash 기준.

## 생존 명령어 10개

### 1. `pwd` — 현재 위치
지금 어느 폴더에 있는지. 길 잃으면 가장 먼저 친다.

### 2. `ls` — 폴더 목록
현재 폴더 내용. `ls -la`는 숨김 파일까지 상세.

### 3. `cd <폴더>` — 폴더 이동
폴더 이름은 `Tab` 키로 자동완성. 예 — `cd Doc<Tab>` → `cd Documents/`.

### 4. `cd ..` — 상위 폴더
점 두 개(`..`)는 '한 단계 위'.

### 5. `cd ~` — 홈으로
어디서든 홈(내 사용자 폴더)으로 돌아온다. 길 잃었을 때.

### 6. `mkdir <이름>` — 새 폴더 만들기

### 7. `touch <파일>` — 빈 파일 만들기
예 — `touch README.md`.

### 8. `cat <파일>` — 파일 내용 보기
짧은 텍스트 파일 출력. 긴 파일에는 부적합.

### 9. `code .` — VS Code로 현재 폴더 열기
점(`.`)은 '현재 폴더'. VS Code 설치되어 있어야 한다.

### 10. `Ctrl+C` — 명령 중단
실행 중인 명령 강제 멈춤. 뭔가 잘못됐을 때 제일 먼저.

## 에러는 정보다

터미널 에러는 무서운 신호가 아니라 단서. 두 종류가 90%를 차지.

- `No such file or directory` — 폴더·파일 이름이 틀렸거나 대소문자 다름. `ls`로 확인 후 다시.
- `command not found` — 도구가 설치 안 됐거나 이름 오타.

**해결 방법은 거의 하나** — 에러 메시지를 통째로 AI에 붙여넣고 묻는다.

```
이 에러가 떴어요: No such file or directory: 'Documents/my-project'
Mac 터미널에서 cd 명령어 쓰고 있어요. 어떻게 해결하나요?
```

이 패턴으로 거의 모든 터미널 에러가 해결된다.

## 시간을 아끼는 4 단축키

| 단축키 | 기능 |
|--------|------|
| `Tab` | 폴더·파일 이름 자동완성 (오타 방지의 핵심) |
| `↑` | 이전 명령어 (재실행할 때) |
| `Ctrl+C` | 실행 중인 명령 중단 |
| `Ctrl+L` | 화면 지우기 |

`Tab` 자동완성을 쓰면 거의 모든 오타가 사라진다.

## 흔한 함정

- **명령어 외우려고 함** — 처음엔 10개만. 나머지는 필요할 때 AI에게 묻는다.
- **에러 보고 멈춤** — 에러는 정보다. 통째로 복사해 AI에 묻는다.
- **공백·대소문자 무시** — 터미널은 둘 다 엄격. `cd Documents`와 `cd documents`는 다르다.
- **`rm -rf` 함부로** — 이건 진짜로 컴퓨터를 망가뜨릴 수 있다. 다음 레슨(git)을 익힌 다음에 손댄다.
