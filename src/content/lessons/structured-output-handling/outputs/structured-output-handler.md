## 1. 출력 스키마 초안

```ts
type AiResult = {
  title: string;
  summary: string;
  confidence: number;
  tags: string[];
  nextActions: string[];
};
```

필수 필드.

- `title`.
- `summary`.
- `confidence`.

선택 필드.

- `tags`.
- `nextActions`.

---

## 2. 성공 응답 예시

```json
{
  "title": "고객 문의 자동 분류",
  "summary": "반복 문의를 4개 유형으로 나눠 첫 답변 초안을 만듭니다.",
  "confidence": 0.82,
  "tags": ["support", "classification"],
  "nextActions": ["최근 문의 20개 수집", "분류 기준 검토"]
}
```

---

## 3. 실패 응답 예시

| 실패 | 예시 | 처리 |
| --- | --- | --- |
| 필드 누락 | `summary` 없음 | 재요청 1회 |
| 타입 불일치 | `confidence`가 `"높음"` | 재요청 1회 |
| 배열 깨짐 | `tags`가 문자열 | 기본값 `[]` 또는 재요청 |
| JSON 아님 | 설명 문단만 반환 | 재요청 1회 후 fallback |

---

## 4. Validation Loop

```txt
1. 모델 응답 수신.
2. JSON 파싱.
3. 스키마 검증.
4. 성공하면 UI로 전달.
5. 실패하면 오류 이유를 포함해 최대 1회 재요청.
6. 다시 실패하면 사용자용 fallback 반환.
7. 실패 원인은 requestId와 함께 로그에 남김.
```

---

## 5. UI fallback

사용자에게 보일 말.

> 결과 형식이 맞지 않아 이번 답변을 표시하지 못했어요. 다시 시도해 주세요.

개발자 로그에 남길 값.

- `requestId`.
- 스키마 실패 경로.
- 원본 응답 일부.
- 재시도 여부.
