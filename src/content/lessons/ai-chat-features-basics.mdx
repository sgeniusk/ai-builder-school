## 같은 기능, 다른 이름

Claude에는 "Projects"가 있다. ChatGPT에도 "Projects"가 있고 "GPTs"도 있다. Gemini에는 "Gems"가 있다. 셋이 같은 건가, 다른 건가.

ChatGPT는 "Memory"를 켜라고 하고, Claude에는 베타로 비슷한 게 있고, Gemini는 또 다르게 한다. Claude에는 "Artifacts"가 있고 ChatGPT에는 "Canvas"가 있다.

이름이 다 다른 게 진짜로 다른 기능인 것은 거의 없다. **공통 6개 기능**의 이름을 한 번 정리해두면, 어떤 챗봇으로 옮겨가도 5분 안에 적응한다.

## 공통 기능 6종

### 1. 작업 공간 (Projects · GPTs · Gems)

대화마다 처음부터 컨텍스트를 다시 설명하지 않게 해주는 묶음. 파일·지시문·이전 대화를 한 곳에 저장해두고, 그 안에서 시작한 대화는 자동으로 그 컨텍스트를 받는다.

- **Claude Projects** — 가장 단순. 파일 업로드 + 지시문 + 대화 묶음.
- **ChatGPT Projects** — 비슷. 추가로 "GPTs"가 있는데, GPTs는 다른 사람과 공유 가능한 정형화된 어시스턴트.
- **Gemini Gems** — 작업별 정형화된 어시스턴트. ChatGPT의 GPTs와 가까움.

**언제 쓰나** — 같은 주제로 반복 대화할 때. 회사 위키 기반 질의, 특정 코드베이스에 대한 작업, 특정 톤의 글쓰기 등.

### 2. 장기 기억 (Memory)

사용자에 대한 사실을 모델이 누적해서 기억한다. 다음 대화에 자동 반영.

- **ChatGPT Memory** — 가장 적극적. 자동으로 사용자의 사실(이름·취향·진행 중인 프로젝트)을 누적.
- **Claude Memory** — 2026 상반기 베타·제한적.
- **Gemini** — 제한적, 발전 중.

**경계해야 할 것** — Memory가 켜져 있는데 민감 정보(주민번호·내부 프로젝트 코드명·고객 이름)를 던지면 그 정보가 다음 대화에 새어 나올 수 있다. 정기적으로 Memory 내용을 확인·삭제하는 습관이 필요.

### 3. CLI · 데스크탑 앱

브라우저 채팅창 밖에서 모델을 쓰는 길.

- **Claude Code** — 터미널에서 동작하는 코딩 에이전트.
- **Claude Desktop** — Mac/Windows 앱. MCP 서버 통합이 강점.
- **ChatGPT Desktop** — Mac/Windows. 데스크탑 통합(스크린샷 · 파일).
- **Codex CLI · ChatGPT iOS Codex** — OpenAI의 코딩·자동화 진입점.

**언제 쓰나** — 코드 작업, 파일 시스템 접근, 외부 도구 호출, 자동화. Stage 4 (코딩 에이전트)에서 본격 다룬다.

### 4. 사이드 캔버스 (Artifacts · Canvas)

대화창 옆에 코드·문서를 따로 그려놓고 반복 수정하는 패널.

- **Claude Artifacts** — 코드·HTML·문서·다이어그램. 미리보기 가능.
- **ChatGPT Canvas** — 코드·긴 글 편집.
- **Gemini Canvas** — 비슷.

**언제 쓰나** — 한 번 답으로 끝나지 않고 같은 산출물을 여러 차례 수정할 때. 글쓰기·디자인·코딩 모두.

### 5. 시스템 지시문 (Custom Instructions)

매 대화에 자동으로 적용되는 사용자 지시. "당신은 항상 한국어로 답하라", "코드 예시에는 항상 주석을 달아라" 같은 글로벌 룰.

- **ChatGPT Custom Instructions** — 두 줄 (당신은 누구인가 + 어떻게 답해주길 원하는가).
- **Claude System Prompt / Project 지시** — Project에 시스템 프롬프트로 박음.
- **Gemini Custom Gem** — Gem 단위로 정의.

**흔한 함정** — 모순된 지시 ("간결하게" + "충분히 자세히"). 모델이 둘 다 만족시키려다 어색해진다. 한쪽으로 정한다.

### 6. 도구·데이터 연결 (MCP · Actions · Plugins)

모델이 외부 도구·데이터(Calendar · Drive · API · 사내 시스템)에 접근하는 길.

- **Claude MCP (Model Context Protocol)** — 표준 프로토콜. 한 번 만든 MCP 서버는 여러 클라이언트에서 재사용. 가장 개방적.
- **ChatGPT GPT Actions** — GPTs 안에서 외부 API 호출 정의.
- **Gemini** — Gmail·Drive 등 Google 생태계 기본 연결, Spark가 확장.

**언제 다루나** — Stage 4·5에서 본격. 지금은 이름만 알아두기.

## 그래서 오늘 할 일

자기가 쓰는 챗봇에서 6개 기능의 위치를 직접 찾는다.

**작업 1 — 위치 찾기 (10분)**

내가 주로 쓰는 챗봇 1개 선택 → 위 6개 기능을 UI에서 직접 찾기. 어디 메뉴에 있는지 메모. 못 찾는 것은 "이 챗봇에는 없음/약함"으로 표시.

**작업 2 — Project + Custom Instructions 만들기 (20분)**

- 자기 일과 가장 관련된 주제로 Project 1개 만들기 (예 "회의록 정리", "주간 보고서")
- Custom Instructions 한 줄 등록 ("답변은 항상 한국어로", "각 주장에 출처를 표기")
- 그 Project 안에서 평소 작업 1개 해보기

**작업 3 — Memory 검토 (10분, Memory 쓰는 사람만)**

- Memory에 어떤 사실이 저장돼 있는지 확인
- 민감 정보가 있으면 삭제
- 향후 Memory에 들어가지 않게 할 정보를 정리

`my-chat-features.md` 한 장 만들어 6개 기능의 위치 + 첫 Project URL을 메모.

## 흔한 함정

- **Memory에 민감 정보 누적** — 가장 자주 나는 사고. 정기 점검 필수.
- **Custom Instructions에 모순된 지시** — "간결 + 충분히 자세히" 같은 충돌. 한쪽으로 정리.
- **Project 컨텍스트 비대화** — 파일을 끝없이 올리면 컨텍스트 윈도우 초과 + 응답 느려짐 + 비용 증가. Project당 핵심 파일만.
