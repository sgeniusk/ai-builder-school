## 1. 테이블 초안

### `sessions`.

| 필드 | 설명 |
| --- | --- |
| `id` | 세션 ID |
| `userId` | 사용자 ID 또는 익명 ID |
| `title` | 사용자가 알아볼 짧은 제목 |
| `createdAt` | 생성 시각 |
| `updatedAt` | 마지막 변경 시각 |

### `messages`.

| 필드 | 설명 |
| --- | --- |
| `id` | 메시지 ID |
| `sessionId` | 연결된 세션 |
| `role` | user / assistant / system |
| `content` | 메시지 본문 |
| `model` | 사용한 모델 |
| `tokenCount` | 추정 또는 실제 토큰 |
| `status` | success / error / cancelled |
| `createdAt` | 생성 시각 |

### `summaries`.

| 필드 | 설명 |
| --- | --- |
| `id` | 요약 ID |
| `sessionId` | 연결된 세션 |
| `coveredMessageIds` | 요약이 덮는 메시지 범위 |
| `summary` | 압축된 맥락 |
| `createdAt` | 생성 시각 |

---

## 2. 저장 흐름

```txt
1. 사용자 메시지를 messages에 저장.
2. 최근 메시지와 최신 summary를 불러옴.
3. AI API 호출.
4. 스트리밍이 끝나면 assistant 메시지를 저장.
5. 실패하면 status와 requestId를 저장.
6. N턴마다 오래된 메시지를 summary로 압축.
7. 다음 호출에는 최신 summary와 최근 메시지만 포함.
```

## 3. 개인정보 마스킹 규칙

| 정보 | 저장 전 처리 | 예외 |
