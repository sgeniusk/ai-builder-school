# AI 코드 전용 PR 체크리스트

일반 코드 리뷰와 다른 점: AI 생성 코드 특유의 패턴에 집중한다.

---

## 필수 항목 (머지 전 모두 체크)

### 환각 API 확인
- [ ] 임포트한 라이브러리/패키지가 실제로 존재하는가
- [ ] 사용한 메서드/함수가 해당 버전 공식 문서에 있는가
- [ ] 파라미터 이름과 타입이 문서와 일치하는가

### Deprecated 패턴 확인
- [ ] 사용한 API가 현재 버전에서 deprecated가 아닌가
- [ ] 프레임워크 마이그레이션 가이드에서 교체된 패턴이 아닌가

### 보안 패턴 확인
- [ ] 하드코딩된 API 키, 비밀번호, 토큰이 없는가
- [ ] SQL 쿼리에 사용자 입력이 직접 삽입되지 않는가 (파라미터화 쿼리 사용)
- [ ] 인증/인가 체크가 필요한 엔드포인트에 적용됐는가
- [ ] 보안 헤더(CORS, CSP 등)가 누락되지 않았는가

### 의도 불일치 확인
- [ ] PR 설명의 요구사항과 실제 코드 동작이 일치하는가
- [ ] Soft delete 요청인데 Hard delete 구현이 아닌가
- [ ] 에러 처리 방식이 요청 명세와 일치하는가
- [ ] 엣지 케이스(NULL, 빈 배열, 최대값 등)가 명세대로 처리되는가

### 코드베이스 일관성
- [ ] 기존 네이밍 컨벤션을 따르는가
- [ ] 기존 에러 처리 패턴과 일관성이 있는가
- [ ] 불필요한 임포트나 미사용 변수가 없는가

## 빠른 참고

| 오류 유형 | 확인 방법 |
|---------|---------|
| 환각 API | 공식 문서 직접 검색 |
| Deprecated | `CHANGELOG.md` 또는 공식 마이그레이션 가이드 |
| 하드코딩 시크릿 | `git grep -i "sk-\|password\|secret\|api_key"` |
| SQL 인젝션 | f-string으로 쿼리 조립 여부 확인 |
| 의도 불일치 | PR 설명 vs 코드 동작 단계별 추적 |
