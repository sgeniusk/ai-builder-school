# LLM 코드 리뷰 프롬프트

AI가 생성한 코드를 AI가 리뷰하는 프롬프트. PR 체크리스트 통과 후 2차 검증으로 사용.

---

## 프롬프트 (복사해서 사용)

```
아래 코드를 다음 항목으로 리뷰해줘.

[리뷰 대상 코드]
<PR 코드 전체 또는 핵심 파일 붙여넣기>

[PR 목적]
<PR 설명 / 요구사항 붙여넣기>

[리뷰 항목]

1. 환각 API
   - 존재하지 않거나 이 버전에서 사용할 수 없는 메서드/파라미터가 있는가?

2. Deprecated 패턴
   - 현재 버전에서 deprecated된 API나 패턴이 사용됐는가?

3. 보안 취약점
   - 하드코딩된 시크릿이나 자격증명이 있는가?
   - SQL 인젝션, XSS 같은 보안 취약 패턴이 있는가?
   - 인증/인가 체크가 누락됐는가?

4. 의도 불일치
   - PR 목적과 실제 코드 동작이 일치하지 않는 부분이 있는가?
   - 요청한 것과 다른 방식으로 구현된 로직이 있는가?

5. 엣지 케이스
   - NULL, 빈 배열, 음수, 최대값 같은 엣지 케이스가 처리됐는가?

[출력 형식]
각 항목별로:
- 문제 없으면: "✅ 이상 없음"
- 문제 있으면: "❌ [라인 번호] [문제 설명] → [수정 방향]"

마지막에 전체 요약 (머지 가능 / 수정 후 머지 / 재작성 권고) 한 줄로.
```

---

## 사용 팁

- 코드가 길면 파일 단위로 나눠 여러 번 리뷰 요청
- "잘 됐나요?" 대신 항목을 명시해야 실질적인 결과가 나옴
- LLM 리뷰가 "✅ 이상 없음"이어도 보안 항목은 정적 분석 도구로 재확인 권장
