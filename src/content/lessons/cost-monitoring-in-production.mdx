## "평소처럼 썼을 뿐인데"

한 달 뒤 청구서를 받고 멈칫합니다. 예상의 4배. 당신이 뭘 잘못한 게 아니에요. 평소처럼 썼습니다. 다만 그사이 자동화 스크립트 하나가 밤마다 루프를 돌았고, 팀원에게 공유한 API 키로 누가 테스트를 반복했고, 이번 달은 마침 바빠서 호출이 늘었어요. 세 가지가 겹쳤는데 — 청구서를 받기 전까지 **아무 신호도 없었습니다.**

AI 비용의 함정은 여기 있어요. **천천히 오르지 않습니다.** 어느 날 한 번에 튑니다.

## 비용이 튀는 3가지 시나리오

### 1. 자동화 폭주
스크립트가 무한 루프에 빠지거나, retry 설정이 5회로 들어가 있거나, cron이 1분마다 돌면 — 호출 수가 천 배가 됩니다.

```text
정상  cron 30분마다 → 1일 48 호출   → 월 1,440  호출 → $7
사고  cron 1분마다  → 1일 1,440 호출 → 월 43,200 호출 → $216
```

같은 코드, 한 줄 설정만 잘못해도 한 달 비용이 30배. 청구서가 한 달 뒤 도착할 때까지 알 수 없어요.

### 2. 키 공유 사고
팀원·동료에게 API 키를 줬는데, 한 명이 본인 사이드 프로젝트에 그대로 박아 두고 잊거나, GitHub public repo에 실수로 올립니다. **봇이 GitHub 공개 키를 1분 안에 찾아냅니다.** Anthropic·OpenAI는 secret scanning으로 자동 무효화하지만 — 무효화되기 전 몇 분 사이에 누군가 다 씁니다.

### 3. 모델 자동 업그레이드
"항상 최신·최고 모델"로 설정해 두면 — 새 모델이 비싸도 자동 전환됩니다. Opus 4.7 vs Flash 3.5는 입력 가격만 16배 차이. 모델 한 줄 바꾸는 PR이 다음 달 청구서를 16배로 만듭니다.

## 알림과 한도 — 다른 일을 한다

대부분의 콘솔이 둘 다 제공하지만, 기본값은 둘 다 꺼져 있어요.

| 장치 | 하는 일 | 위치 |
|---|---|---|
| **사용량 알림** (Usage Alert) | 예산의 50/80/100% 도달 시 이메일·문자 | 콘솔 Settings |
| **지출 한도** (Hard Limit) | 그 금액 넘으면 호출 자체를 차단 | 콘솔 Billing |

알림만 있으면 — 메일 못 봤으면 끝. 한도만 있으면 — 갑자기 서비스 죽음. **둘 다 걸어야 안전합니다.**

## OpenAI · Anthropic · Google — 어디서 거나

각 콘솔의 위치만 짚어 둡니다. 본인 쓰는 도구에 들어가 5분이면 둘 다 끄고 켤 수 있어요.

- **OpenAI** (platform.openai.com) — Billing → **Usage limits** → Hard limit + Soft limit
  - 권장 시작점 — Hard $50, Soft $30 (개인 학습)
- **Anthropic** (console.anthropic.com) — Settings → **Spend limits** (Workspace별, API 키별 사용량 분리 추적)
- **Google AI** (aistudio.google.com → Cloud Console) — Billing → **Budgets & alerts** → 50/80/100% + Hard limit
- **Cursor·Claude Code 정기구독** — 콘솔이 아니라 카드 결제 한도로 통제. 카드사 앱에서 월 한도 설정

## 매주 5분 점검 — 무엇을 보나

매주 같은 요일·같은 시간 — 5분이면 끝납니다.

1. **이번 주 사용량** — 콘솔 대시보드 한 화면
2. **지난 주 대비** — 2배 이상 증가했나? 왜?
3. **가장 비싼 호출** — 모델별·작업별 톱 3. 패턴이 보임
4. **새 키 생성 없었나** — 본인이 안 만든 키가 있나
5. **다음 주 예상** — 진행 중인 자동화·실험이 비용을 늘릴까

캘린더에 "AI 비용 점검" 5분 슬롯 — **이게 핵심**. 안 정해두면 안 봅니다. 매주 월요일 오전 9시처럼 고정.

## 청구서가 이미 터졌다면

당황하지 말고 5분 안에 4 단계. **출혈을 먼저 멈춰야 분석할 시간이 생깁니다.**

1. **출혈 멈춤** — 한도 0으로 / 키 회수 / 자동화 스크립트 kill. 서비스 죽어도 비용이 우선
2. **로그 확인** — 콘솔 → API logs로 어떤 키·어떤 호출 패턴이었는지
3. **원인 분류** — 자동화 폭주 / 키 노출 / 정상 트래픽 증가
4. **대응** — 자동화면 코드 수정 후 한도 재개. 키 노출이면 회수 + 새 키 + Git history에서 제거 (`git filter-repo`)

> 청구서 폭탄을 막지 못한 경우, 콘솔에서 **"unauthorized usage"** 같은 명목으로 일부 환불을 요청할 수 있어요. 사후 대응 절차에 환불 요청 단계도 포함.

## 그래서 오늘 할 일

오늘 만들 한 장은 `cost-monitor.md`입니다. 본인이 실제로 쓰는 AI 도구를 목록으로 만들고, 각각에 알림과 한도를 걸고, 매주 5분 점검 루틴을 캘린더에 박는 거예요. 빌드 단계의 미션을 따라가세요.

## 흔한 함정

- **"내 사이드 프로젝트는 작아서 한도 필요 없어"** — 가장 자주 청구서 폭탄 맞는 사람들의 말. 작아도 한도는 무조건.
- **알림만 걸기** — 메일 안 보면 끝. 한도까지 같이.
- **GitHub에 키 공개** — secret scanning이 막아주지만, 막히기 전 몇 분 사이 봇이 다 씁니다. `.env` + `.gitignore` 반드시.
- **팀에 단일 키 공유** — 누가 얼마 썼는지 못 봅니다. 키는 사람당 1개.
