# 서술 생성 프롬프트 템플릿

`[KPI 데이터]` 와 `[청중]` 만 바꿔서 재사용한다.

---

## 기본 프롬프트

```
[KPI 데이터]
기간: <YYYY년 MM월>
- <지표명>: <수치> (<전월 수치>, <+/-변화량 or %>)
- <지표명>: <수치> (<전월 수치>, <+/-변화량 or %>)
- <지표명>: <수치> (<전월 수치>, <+/-변화량 or %>)

[청중]
경영진 / 팀 리뷰 (해당 항목 선택)

[서술 형식]
아래 구조로 작성해줘:

**사실 서술** (위 데이터에서 직접 읽을 수 있는 것만, 추측 금지):
- ...

**가설 서술** (원인 또는 해석, 반드시 "(가설)" 표시, 근거 없으면 쓰지 말 것):
- ...

**다음 확인이 필요한 것**:
- ...

[제약]
- 근거가 없는 원인 설명은 쓰지 마세요
- 가설은 반드시 "(가설: )" 형식으로 표시하세요
- 경영진용이면 3문장 이내, 팀 리뷰용이면 항목별 2~4문장
- 수치 계산은 내가 준 데이터 기준으로만 하세요
```

---

## 경영진 보고용 추가 지시

```
경영진 보고용 3문장 요약 버전도 작성해줘:
1문장: 이번 달 가장 주목할 변화
2문장: 그 맥락 또는 배경 (데이터 기반만)
3문장: 이번 주 확인이 필요한 행동 하나
```

---

## 이상치 탐지 프롬프트

```
위 KPI 중 전월 대비 변화가 큰 이상치를 찾아줘.
- 전월 대비 ±10% 이상 변화한 지표를 나열해줘
- 각 이상치에 대해 "추가 확인이 필요한 질문" 1개씩 제안해줘 (원인을 추측하지 말고, 확인해야 할 것을 질문 형식으로)
```
