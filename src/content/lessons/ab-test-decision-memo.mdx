## p-value 0.03이 나왔다. 배포해야 할까?

그냥 물어보면 AI는 이렇게 답한다.

> *"p-value가 0.05 미만으로 통계적으로 유의미합니다. B 버전 배포를 권장합니다."*

여기서 멈춰야 한다.

샘플 크기는 충분했는가? 테스트 기간은 적절했는가? 전환율 차이가 실제 비즈니스에 의미 있는 크기인가? AI는 이런 질문을 먼저 하지 않는다.

---

## 두 가지 유의미성

**통계적 유의미성 (Statistical Significance):** p < 0.05는 '이 결과가 우연일 확률이 5% 이하'라는 의미다. 방향이 맞다는 뜻이지, 크기가 중요하다는 뜻이 아니다.

**실용적 유의미성 (Practical Significance):** 전환율이 3.20%에서 3.21%로 올랐어도 p < 0.05가 나올 수 있다. 그 0.01%p가 비즈니스에 의미 있는가?

AI에게 결과만 주고 '해석해줘'라고 하면 통계적 유의미성만 보고 결론을 낸다. 실용적 유의미성은 내가 물어야 나온다.

---

## AI의 낙관적 해석 편향

A/B 테스트 결과를 해석할 때 AI는 긍정적인 결론 쪽으로 기운다. 왜냐하면 학습 데이터에서 '성공한 실험' 케이스가 많기 때문이다.

이 편향을 줄이는 방법이 있다. **제약 사항을 먼저 물어라.**

**일반 요청:**
```
버튼 색상 A/B 테스트 결과:
- A: 전환율 4.1%, 샘플 300명
- B: 전환율 4.7%, 샘플 300명
- p-value: 0.038

이 결과를 해석해줘.
```

**제약 사항 우선 요청:**
```
버튼 색상 A/B 테스트 결과:
- A: 전환율 4.1%, 샘플 300명
- B: 전환율 4.7%, 샘플 300명
- p-value: 0.038, 테스트 기간: 5일

먼저 이 결과를 신뢰하기 어려운 이유 3가지를 말해줘.
그다음에 배포 / 보류 / 재실험 중 권고 사항을 말해줘.
```

두 번째 프롬프트에서 AI는 샘플 크기 부족, 테스트 기간 짧음, 노벨티 효과 가능성 같은 제약을 먼저 언급한다.

---

## 자주 놓치는 제약 사항

| 항목 | 확인 질문 |
|------|---------|
| 샘플 크기 | 충분한 통계 파워를 얻을 만큼의 샘플인가? (일반적으로 각 그룹 최소 1,000명 권장) |
| 테스트 기간 | 최소 1~2주기의 트래픽 패턴이 포함됐는가? (주말/평일 차이 반영) |
| 노벨티 효과 | 새 버전을 처음 봤을 때 일시적으로 클릭이 높은 것은 아닌가? |
| 다중 비교 | 여러 지표를 동시에 보고 있다면 Bonferroni 보정이 필요하다 |
| 세그먼트 편향 | 모바일/데스크톱, 신규/기존 사용자 비율이 두 그룹에서 같은가? |

이 항목들을 AI에게 직접 물어보거나, 프롬프트에 수치를 주면서 평가를 요청한다.

---

## 1페이지 의사결정 메모 형식

통계 리포트와 다르다. 의사결정 메모는 **다음 행동에 집중**한다.

```
## A/B 테스트 의사결정 메모
테스트명: 온보딩 버튼 색상 (파란색 vs 초록색)
기간: 2025-04-15 ~ 2025-04-20 (5일)

### 결과 요약
- A (파란색): 전환율 4.1%, n=300
- B (초록색): 전환율 4.7%, n=300
- 절대 차이: +0.6%p | 상대 차이: +14.6%
- p-value: 0.038 (유의수준 0.05 기준 통과)

### 제약 사항
- 샘플 크기: 각 300명 — 검출 가능 최소 효과 크기(MDE) 기준 부족 가능성
- 테스트 기간: 5일 — 주말 트래픽 패턴 미반영
- 노벨티 효과 미통제

### 결론
**보류** — 재실험 권고

### 다음 행동
1. 샘플 크기 계산: 전환율 1%p 차이 감지를 위한 필요 샘플 수 계산
2. 테스트 기간 연장: 최소 14일, 주말 2회 포함
3. 세그먼트 분리 분석: 모바일 vs 데스크톱 결과 분리 확인
```

'배포 / 보류 / 재실험' 중 하나를 명확히 쓴다. 애매한 결론은 결론이 아니다.

---

## AI가 통계 계산을 도울 때

샘플 크기 계산, 통계 파워 계산은 AI가 도움이 된다.

```
현재 기준선 전환율이 4.1%일 때,
최소 1%p 차이(절대값)를 95% 신뢰수준, 80% 파워로 감지하려면
각 그룹에 몇 명이 필요한지 계산해줘.
```

단, AI의 계산 결과는 직접 검증한다. 계산식이나 파이썬 코드로 재현 가능한지 확인하는 것이 안전하다.

## 핵심 원칙

> 통계적으로 유의미하다는 것이 배포해야 한다는 의미가 아니다.
> AI에게 강점이 아니라 약점을 먼저 물어라.
> 결론은 명확해야 한다. 배포, 보류, 재실험 중 하나.

A/B 테스트 해석에서 AI는 보조 도구다. 최종 판단은 비즈니스 맥락을 아는 사람이 한다.
