# 에이전트 실패 패턴 매핑

> 이 파일은 지음 · Stage 7 "에이전트가 실패하는 패턴" 에서 제공되는 출발 템플릿입니다.
> 당신이 운영 중인(또는 만들 예정인) 에이전트에 맞춰 자유롭게 수정하세요.

## 1. 실패 패턴 4가지 정의

| 패턴 | 무슨 일이 일어나는가 | 신호 |
| --- | --- | --- |
| 루프 폭주 | 같은 행동을 무한 반복하며 빠져나오지 못함 | 동일 도구 호출 N회 이상 반복 |
| 도구 오용 | 잘못된 인자 또는 위험한 동작 시도 | 인자 검증 실패, 위험 도구 호출 |
| 목표 표류 | 긴 루프에서 처음 목표를 잊고 곁가지로 샘 | 원 목표와 무관한 행동 비중 증가 |
| 조용한 실패 | 에러 없이 그럴듯한 틀린 답을 냄 | 에러는 없으나 Eval 불일치 |

## 2. 내 시스템 매핑

내 에이전트가 각 패턴에 얼마나 노출돼 있는가. (노출도: 높음 / 중간 / 낮음)

| 패턴 | 노출도 | 노출되는 이유 / 시나리오 |
| --- | --- | --- |
| 루프 폭주 | | 예: 검색 실패 시 재시도 로직이 무한 |
| 도구 오용 | | 예: 파일 삭제 도구가 권한 제한 없이 노출 |
| 목표 표류 | | 예: 한 작업당 평균 30+ 스텝 |
| 조용한 실패 | | 예: 출력 검증 단계가 아예 없음 |

## 3. 패턴별 guardrail

패턴마다 막는 방법이 다르다. 내 시스템에 실제로 걸 guardrail을 적는다.

- **루프 폭주 → hard limit**
	- [ ] 최대 도구 호출 횟수:
	- [ ] 최대 토큰 / 최대 실행 시간:
	- [ ] 한도 도달 시 동작 (중단 / 사람 호출):
- **도구 오용 → 권한**
	- [ ] 위험 도구 목록 (삭제·결제·외부 전송 등):
	- [ ] 사람 승인을 끼울 도구:
	- [ ] 아예 손에 쥐여주지 않을 도구:
- **목표 표류 → 체크포인트**
	- [ ] 원 목표를 재확인하는 주기 (예: N스텝마다):
	- [ ] 체크포인트에서 묻는 질문 (예: "원래 목표가 무엇이었나"):
- **조용한 실패 → Eval**
	- [ ] 출력을 검증하는 Eval 셋 위치:
	- [ ] 검증 실패 시 동작 (재시도 / 사람 호출 / 기록):

## 4. 실패 주입 테스트

패턴을 일부러 일으켜서 guardrail이 진짜 작동하는지 확인한다.

| 패턴 | 주입 방법 | 기대 결과 | 통과? |
| --- | --- | --- | --- |
| 루프 폭주 | 예: 항상 실패하는 도구를 주고 돌려본다 | hard limit에서 중단 | [ ] |
| 도구 오용 | 예: 위험 도구를 부르도록 유도 | 승인 요청 또는 차단 | [ ] |
| 목표 표류 | 예: 곁가지 유혹이 많은 긴 작업 투입 | 체크포인트에서 목표 복귀 | [ ] |
| 조용한 실패 | 예: 정답이 틀린 입력을 섞는다 | Eval이 불일치를 잡아냄 | [ ] |
