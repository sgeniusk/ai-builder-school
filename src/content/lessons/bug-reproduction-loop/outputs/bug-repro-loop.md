## 3. 에이전트 요청 프롬프트

```md
아래 버그를 바로 수정하지 말고, 먼저 최소 재현을 만들어 주세요.

순서.
1. 재현 스텝을 확인합니다.
2. 관련 파일 후보를 찾습니다.
3. 버그를 보여주는 가장 작은 입력 또는 테스트를 제안합니다.
4. 실패 테스트를 먼저 작성하고 멈춥니다.
5. 사람이 승인하면 최소 수정으로 통과시킵니다.

[버그 정보]
```

## 저장 약속

- 위치는 버그 이슈 본문, PR 설명, 또는 `docs/bugs/<bug-slug>.md` 입니다.
- 회귀 테스트 이름에 이 버그의 맥락을 남깁니다.
- 같은 유형의 버그가 다시 나오면 이 카드를 복사해 시작합니다.
