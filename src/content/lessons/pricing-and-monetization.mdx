## 무료가 기본값이 되면 생기는 문제

베타 사용자 50명이 생겼다. 근데 무료다.

API 비용은 매달 나간다. 수익은 없다. 유료 전환을 시도하면 "왜 갑자기 내야 해?" 라는 반응이 온다.

처음부터 가격 구조를 설계하지 않으면 무료 사용자를 쌓아두고 전환 못 하는 함정에 빠진다.

---

## AI 제품의 비용 구조 먼저

가격을 정하기 전에 "내가 얼마나 나가는지"를 안다.

```
사용자 1명이 하루 10회 호출 시:
- Claude Sonnet 기준: 약 $0.03/일
- 30일 기준: 약 $0.90/월

사용자 100명이면:
- 월 $90 API 비용
```

가격 = API 비용 × 사용량 + 마진. 마진이 없으면 사업이 아니다.

---

## 3가지 모델 비교

### Freemium

무료 플랜 → 한도 도달 → 유료 전환.

**장점:** 사용자 유입이 빠름  
**단점:** 무료 한도 설계 실패 시 전환율 0%

```
무료: 월 5회 요약
유료: 월 무제한 + 팀 공유
```

무료 한도는 "가치를 느끼기엔 충분하고, 쓰기엔 부족한" 수준이어야 한다.

### 구독 (Subscription)

월 정액. 단순하고 예측 가능하다.

**장점:** 현금 흐름 안정, 가격 이해가 쉬움  
**단점:** 사용량이 적은 달에도 동일 금액 → 해지 유발

```
스탠다드: $9/월
프로: $29/월 (팀 기능 + 우선 지원)
```

### 사용량 기반 (Pay-per-use)

쓴 만큼 낸다.

**장점:** 초기 사용자에게 진입 장벽 낮음  
**단점:** 청구서 예측 어려움 → 사용자 불안

```
요약 1건당 $0.05
월 최대 $20 (캡 설정)
```

---

## 업그레이드 트리거 설계

사용자가 "이제 돈 내야겠다"고 느끼는 순간을 의도적으로 만든다.

| 트리거 유형 | 예시 |
|------------|------|
| 한도 초과 | "이번 달 5회 사용을 다 썼습니다. 계속하려면 업그레이드하세요." |
| 고급 기능 | "팀 공유는 프로 플랜에서 사용 가능합니다." |
| 속도/품질 | "프로 플랜에서 2배 빠른 모델을 사용합니다." |

트리거가 없으면 무료 사용자는 영원히 무료로 쓴다.

## 가격 책정 실습

**Step 1.** AI에게 비용 데이터를 주고 시뮬레이션 요청:

```
내 AI 제품의 비용 구조야:
- 사용자 1명 평균 월 [N]회 호출
- 1회 호출 평균 비용 $[X]

이 데이터로 Freemium / 구독 / 사용량 기반
세 모델의 가격을 제안하고, 각 모델의 예상 마진을 계산해줘.
월 사용자 100명 기준으로.
```

**Step 2.** 마진 계산:

```
구독 $9/월 × 100명 = $900 수입
API 비용 $0.90/명 × 100명 = $90
마진 = $810 (90%)
```

**Step 3.** 페르소나에게 가장 납득되는 모델 선택.

**Step 4.** 3플랜 설계:

```
무료: [가장 기본 기능 + 한도]
스탠다드: [핵심 기능 전체]  $X/월
프로: [고급 기능 + 팀 기능]  $Y/월
```

## 처음 유료 전환 만들기

100명 중 1명에게 먼저 물어본다:

```
"[이름]님, 지금까지 무료로 써주셔서 감사해요.
저희가 유료 플랜($9/월)을 출시할 예정인데,
제일 먼저 써보실 의향 있으신가요?
한 달 무료로 드릴게요."
```

직접 요청이 이메일 폼보다 전환율이 높다.

## 핵심 원칙

> 가격은 내 비용이 아니라 사용자의 가치 기준으로 설정한다.
> 업그레이드 트리거를 미리 설계하지 않으면 무료 사용자는 유료로 전환하지 않는다.
