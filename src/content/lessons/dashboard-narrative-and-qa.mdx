## 숫자는 말하지 않는다

전환율이 3.2%에서 2.8%로 떨어졌다. 이게 전부인가?

경영진에게는 아니다. 그들은 '왜'와 '그래서 뭘 해야 하는지'를 원한다.

KPI 대시보드의 숫자를 서술로 바꾸는 일이 여기서 시작된다. 문제는 이 작업을 AI에게 맡기면 AI가 **근거 없는 원인을 만들어낸다**는 점이다.

---

## AI가 숫자를 해석할 때 생기는 문제

아래 프롬프트를 그냥 쓰면 위험하다.

```
이번 달 KPI:
- MAU: 124,000 (전월 131,000)
- 전환율: 2.8% (전월 3.2%)
- 평균 주문금액: 42,000원 (전월 41,200원)

이 데이터를 분석해줘.
```

AI의 전형적인 응답:
> *"MAU 감소는 경쟁사 신규 서비스 출시의 영향으로 보입니다. 전환율 하락은 모바일 UX 개선이 필요한 시점임을 시사합니다..."*

이 문장들의 근거는 내가 준 데이터에 없다. AI가 지어낸 것이다.

---

## 사실 서술 vs 해석 서술 분리

프롬프트에서 두 가지를 명시적으로 분리하면 이 문제가 줄어든다.

```
[KPI 데이터]
MAU: 124,000 (전월 131,000, -5.3%)
전환율: 2.8% (전월 3.2%, -0.4%p)
평균 주문금액: 42,000원 (전월 41,200원, +1.9%)

아래 형식으로 서술해줘:

**사실 서술** (데이터에서 직접 읽을 수 있는 것만):
- ...

**가설 서술** (원인 또는 해석, 반드시 "(가설)" 표시):
- ...

**다음 확인이 필요한 것**:
- ...
```

이렇게 하면 AI가 근거 있는 것과 추측을 섞지 않고 분리한다.

---

## 근거 추적 QA

AI가 생성한 서술을 그대로 쓰지 않는다. 각 문장마다 묻는다.

> **이 주장의 근거가 내가 준 데이터에 있는가?**

| 서술 | 근거 있음 | 처리 |
|------|---------|------|
| "MAU가 5.3% 감소했습니다" | ✅ 데이터 직접 계산 | 유지 |
| "전환율이 0.4%p 하락했습니다" | ✅ 데이터 직접 계산 | 유지 |
| "모바일 UX가 원인입니다" | ❌ 데이터 없음 | 삭제 또는 (가설) 표시 |
| "경쟁사 출시의 영향" | ❌ 데이터 없음 | 삭제 또는 (가설) 표시 |

근거 없는 원인 규명은 보고서에서 가장 위험한 부분이다. 경영진이 그 가설을 사실로 받아들이고 의사결정할 수 있다.

---

## 청중 맞춤 서술

같은 KPI도 청중에 따라 다른 서술이 필요하다.

**경영진 보고용 (3문장 이내, 행동 지향):**
```
[청중] 경영진 (비기술, 의사결정자)
[목적] 다음 행동 결정
[길이] 3문장 이내
[형식] 현황 → 주목할 변화 → 권고 행동
```

**팀 리뷰용 (상세, 데이터 중심):**
```
[청중] 데이터 팀 / 제품팀
[목적] 원인 분석 + 다음 단계 정의
[길이] 항목별 2~4문장
[형식] 수치 → 전월 비교 → 이상치 → 가설 → 확인 필요 항목
```

같은 프롬프트로 두 버전을 요청하고, 경영진용은 팀 리뷰용보다 더 짧고 행동 지향적으로 나오는지 확인한다.

---

## 서술 구조 고정

매 리포트마다 AI가 다른 형식으로 서술하면 일관성이 없다. 구조를 프롬프트에 고정한다.

```
아래 구조로 월간 KPI 코멘트를 작성해줘:

1. **이번 달 핵심 (1문장)** — 가장 주목할 변화 하나
2. **지표별 현황** — 각 KPI의 수치 + 전월 대비 (사실만)
3. **이상치** — 정상 범위에서 크게 벗어난 지표와 그 크기
4. **가설** (표시 필수) — 원인 추정 (근거 있는 데이터 인용 또는 "(확인 필요)" 표시)
5. **다음 행동** — 이번 주 확인해야 할 것 2~3가지
```

이 구조를 매달 같은 템플릿으로 쓰면 리포트 형식이 통일된다.

---

## AI 코멘트 QA 체크리스트

서술 생성 후 이 체크리스트를 돌린다.

- [ ] 사실 서술과 가설 서술이 분리되어 있는가
- [ ] 가설에 "(가설)" 또는 "(확인 필요)" 표시가 있는가
- [ ] 원인 규명 문장의 근거를 데이터에서 추적할 수 있는가
- [ ] 경영진용 서술이 3문장 이내이고 행동 지향적인가
- [ ] 이상치가 누락 없이 언급됐는가
- [ ] 수치가 내가 준 데이터와 일치하는가 (AI가 계산 틀릴 수 있음)

마지막 항목이 중요하다. AI가 퍼센트 계산을 틀리는 경우가 있다. 수치는 반드시 원본 데이터와 대조한다.

## 핵심 원칙

> 숫자는 사실이다. 원인은 가설이다. AI는 이 둘을 섞는다.
> 근거 없는 원인 규명은 삭제하거나 가설로 표시한다.
> 수치 계산은 AI가 틀릴 수 있다. 항상 원본과 대조한다.
