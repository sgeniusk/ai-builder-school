## AI Builder School에서 배우는 것

코딩을 배우는 곳이 아니다.

AI를 **지시하고 검토하는 법**을 배우는 곳이다.

이 차이가 중요하다. 코딩 에이전트는 코드를 짠다. 사람은 방향을 정하고, 결과를 검토하고, 다음 지시를 내린다. 의사가 MRI 기계 원리를 몰라도 영상을 보고 판단하듯이, AI Builder School을 마치면 코드 원리를 몰라도 AI로 무언가를 만들 수 있다.

---

## 코딩 에이전트 vs 일반 AI 챗봇

| | 일반 AI 챗봇 (ChatGPT 웹, Claude.ai) | 코딩 에이전트 (Codex, Claude Code) |
|---|---|---|
| 하는 일 | 대화, 설명, 텍스트 생성 | 파일 생성, 코드 실행, 에러 수정 |
| 결과물 | 텍스트 답변 | 실제 파일, 실행 가능한 코드 |
| 작업 방식 | 내가 복사해서 써야 함 | 직접 파일에 씀 |
| 비유 | 조언해주는 친구 | 실제로 일하는 직원 |

챗봇은 "이렇게 하면 돼요"라고 말한다. 코딩 에이전트는 실제로 한다.

---

## 터미널이 왜 필요한가

코딩 에이전트는 **터미널을 통해 실행**된다.

클릭이 아니라 텍스트 명령으로 컴퓨터에 지시하는 방법이 터미널이다. 겁먹지 않아도 된다. 코딩 에이전트를 쓰기 위해 필요한 명령어는 10개 이내다.

```
codex "로그인 페이지를 만들어줘"
```

이 한 줄로 코딩 에이전트가 시작된다. 그 뒤는 에이전트가 한다.

---

## AI-사람 협업 구조

```
나 → 목표와 방향 설정
AI → 코드 작성, 파일 생성, 에러 수정
나 → "이게 내가 원하는 건가?" 판단
나 → 다음 지시
AI → 반복
```

이 루프가 AI Builder School의 핵심이다. 코드를 이해하는 게 아니라 루프를 잘 돌리는 법을 배운다.

## 12개 Phase 전체 지도

| Phase | 주제 | 이후에 할 수 있는 것 |
|-------|------|------------------|
| 0 | 환경 세팅 | 도구를 켜고 쓸 수 있다 |
| 1 | AI 리터러시 | AI를 비판적으로 쓴다 |
| 2 | 프롬프트 엔지니어링 | AI에게 정확히 지시한다 |
| 3 | 일상 자동화 | 반복 업무를 AI에게 맡긴다 |
| 4 | 코딩 에이전트 | AI와 함께 코드를 만든다 |
| 5 | AI 앱 개발 | API를 연결한 서비스를 만든다 |
| 6 | RAG | 내 문서로 AI를 강화한다 |
| 7 | 에이전트 & MCP | 자율로 동작하는 에이전트 |
| 8 | 멀티모달 & 콘텐츠 | 영상·이미지·글 자동화 |
| 9 | 데이터 분석 | 숫자를 판단으로 바꾼다 |
| 10 | Evals & 보안 | AI 시스템을 안전하게 운영한다 |
| 12 | 캡스톤 | 실제 프로젝트를 완성한다 |

모든 걸 다 할 필요는 없다. 내 목표에 맞는 Phase만 골라도 된다.
