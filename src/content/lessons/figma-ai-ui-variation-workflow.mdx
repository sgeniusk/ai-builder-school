## 버튼 하나 만들었더니 변형이 72개 — 한나절이 사라져요

버튼 컴포넌트 1개를 만들었어요. 깔끔하게 잘 됐어요. 그런데 디자인 시스템에는 Primary/Secondary/Ghost × Small/Medium/Large × Default/Hover/Active/Disabled × Light/Dark 가 필요합니다. 곱하면 *3 × 3 × 4 × 2 = 72개* 변형. 손으로 다 만들면 한나절. 그리고 다음 컴포넌트(인풋, 카드)도 똑같은 매트릭스를 처음부터 다시.

AI 에게 *"Figma 버튼 변형 만들어줘"* 하면 React/CSS 코드를 줘요. 근데 그건 Figma 에 못 옮겨요. 토큰 일관성도 깨지고, 다크모드 대비비도 안 맞고. 그래서 결국 손으로 다시 만듭니다.

오늘은 *손으로 그리기* 대신 *매트릭스로 관리* 하는 5단계 워크플로우를 만들어요. 변형 축 → 변형 규칙(토큰) → AI 명세 생성 → Figma 반영 → 접근성 검증.

## 변형 축 — 변형 개수를 곱셈으로 계산하기

먼저 컴포넌트가 가질 수 있는 *변형의 차원* 을 명시해요. 보통 4축이에요.

| 축 | 값 | 개수 |
| --- | --- | --- |
| 사이즈 | Small / Medium / Large | 3 |
| 상태 | Default / Hover / Active / Disabled | 4 |
| 테마 | Light / Dark | 2 |
| 변종 | Primary / Secondary / Ghost | 3 |

곱하면 *3 × 4 × 2 × 3 = 72개*. 즉흥적으로 "이것도 필요하네" 하면서 만들지 않아요. 축을 정하면 *전체 그림* 이 보입니다.

## 변형 규칙 — 손으로 그리지 않고 토큰으로

각 변형을 *손으로 그리는 게 아니라 규칙으로 정의* 해요.

- 사이즈 Small → 패딩 `space-2` (8px), 폰트 `text-sm` (14px), 반경 `radius-1`
- 상태 Disabled → opacity `0.4`, 커서 `not-allowed`
- 테마 Dark → 배경 `surface-dark`, 텍스트 `text-on-dark`

이 규칙이 *디자인 토큰* 에 묶이면, 토큰 하나만 바꿔도 *전체 변형이 따라옵니다*. `space-2` 를 8px → 10px 으로 바꾸면 모든 Small 변형이 자동으로. 이게 디자인 시스템의 핵심이에요.

## 상태 매트릭스 — 빈 셀이 한눈에

변형 축을 *행 × 열 표* 로 펼쳐요.

| | Default | Hover | Active | Disabled |
| --- | --- | --- | --- | --- |
| Primary / S / Light | ✓ | ✓ | ✓ | ✓ |
| Primary / M / Light | ✓ | ✓ | ✓ | (빈칸) |
| Secondary / S / Dark | ✓ | (빈칸) | ✓ | ✓ |

각 셀이 *하나의 변형* 이에요. AI 에게 이 매트릭스를 입력하면 각 셀의 *토큰 값과 스펙* 을 자동으로 채워줍니다. 빈 셀 = 아직 안 만든 변형이 한눈에 보여요. *빈 셀이 0개* 가 되면 완성.

## AI ↔ Figma 왕복 — 명세가 단일 진실 원천

AI 가 Figma 를 직접 조작하긴 어려워요. 대신 이렇게 왕복합니다.

```
[AI] 변형 명세 생성 (Property 이름 + Variant 조합 + 토큰 표)
        ↓
[디자이너] Figma Variants/Properties 로 옮김
        ↓
[디자이너] 실제로 보면서 명세 업데이트 ("Hover 그림자 더 부드럽게")
        ↓
[AI] 업데이트된 명세로 나머지 변형 재생성
```

*명세(spec)가 단일 진실 원천(SSOT)* 이 돼요. Figma 도 명세를 따르고, 나중에 코드로 export 할 때도 명세를 따릅니다. 디자인-개발 사이의 어긋남이 줄어요.

## 접근성 검증 — AI 가 자주 틀리는 곳

AI 가 변형을 만들 때 *대비비* 를 자주 틀려요. 특히 다크모드. Ghost 버튼의 텍스트가 배경과 대비 4.5:1 이 안 되는 경우가 흔합니다. 그래서 모든 변형을 체크리스트로 검증해요.

- [ ] 텍스트 대비비 4.5:1 이상 (큰 텍스트는 3:1)
- [ ] 터치 타깃 최소 44 × 44px (모바일)
- [ ] 포커스 링 가시성 (키보드 탐색)
- [ ] Disabled 상태도 *읽을 수는* 있게 (완전 투명 X)

이걸 빠뜨리면 예쁜데 못 쓰는 컴포넌트가 됩니다.

## 오늘 할 일 — 50분 안에 컴포넌트 1개를 매트릭스로

본인의 *실제 컴포넌트 1개* (버튼/인풋/카드 중 하나) 를 골라 한 번 끝까지 돌려 봅니다. `ui-variation-workflow/` 폴더에 5 파일을 만들고, 마지막에 *상태 매트릭스의 빈 셀이 0개* 가 되도록 채워보세요. 미션 박스의 6 단계가 시간 가이드입니다.

끝나고 나면 *한나절 걸리던 변형 작업* 이 *한 시간* 으로 줄어요. 그리고 이 워크플로우는 다음 컴포넌트(인풋, 카드)에도 *호출* 만 하면 돼요. 한 번 만든 매트릭스 구조가 *자산* 이 됩니다.
