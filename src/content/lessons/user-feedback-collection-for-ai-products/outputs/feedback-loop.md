내부 Eval이 검사하는 기준과, 그 바깥에 있는 사용자 기대를 나란히 적는다.

| 내 Eval이 검사하는 것 | Eval이 못 보는 사용자 기대 (가설) |
| --- | --- |
| 예: 답변 정확도 | 예: 답변이 너무 길어서 안 읽음 |
| 예: 톤이 정중한가 | 예: 원한 건 요약인데 설명을 받음 |

## 3. 비즈니스 지표 3개

품질 점수 대신, 제품이 정말 잘되는지 보여줄 지표를 고른다.

| 지표 | 정의 (어떻게 측정) | 현재 값 | 목표 값 |
| --- | --- | --- | --- |
| 예: 7일 재방문율 | 첫 사용 후 7일 내 다시 온 비율 | % | % |
| 예: 작업 완료율 | 시작한 작업을 끝까지 마친 비율 | % | % |
| 예: 30일 유지율 | 30일 후에도 쓰는 사용자 비율 | % | % |

- 회고 요일·시간 (캘린더 고정 슬롯):
- 회고 때 보는 순서
	1. 이번 주 좋아요 / 싫어요 비율
	2. 자유 코멘트 중 반복되는 말 1~2개
	3. 비즈니스 지표 3개가 지난주 대비 어떻게 움직였나
	4. 이 중 다음 주에 고칠 한 가지를 정한다
