## "사람이 매 단계 검토하던 LLM 코딩이 /goal 한 줄로 자율 루프가 됐어요"

기존 LLM 코딩은 매 단계 사람이 끼어들어요. 인간이 입력하면 AI가 응답하고, 다음을 진행하려면 또 사람이 검토하고 새 지시를 줘야 합니다. 매 단계 사람의 손이 필요한 거예요.

OpenAI Codex CLI 0.128.0에서 등장한 `/goal` 기능은 이 패러다임을 바꿉니다. **목표만 주면 진짜 스스로 코딩**해요. Anthropic도 비슷한 방향으로 가고 있고, 이 흐름의 원형이 **Ralph Loop**입니다.

[Lesson 28](/lessons/mini-agent-loop)에서 미니 에이전트 루프의 일반론을 다뤘다면, 오늘은 그 루프가 **상용 도구의 공식 기능으로 진화**한 사례를 들여다봅니다.

## Ralph Loop — 자율 루프의 원형

{/* Source · DOC · Geoffrey Huntley "Ralph Wiggum as a Software Engineer" — https://ghuntley.com/ralph/ */}
{/* Note: "Ralph Loop" 명칭의 유래는 Huntley의 5줄짜리 bash 스크립트와 관련되며, 본문의 진화 단계 분류는 이 학교 운영자의 정리입니다. */}

Ralph Loop은 Geoffrey Huntley가 처음 공개한 5줄짜리 bash 스크립트에서 시작했어요. 핵심 아이디어는 한 줄로 정리됩니다.

> 같은 프롬프트를 끝없이 반복해서, AI가 스스로 진척을 만들게 하라.

이 단순한 아이디어가 3단계로 진화해요.

| 단계 | 무엇 | 누가 |
| --- | --- | --- |
| **1세대 — 단순 반복** | 같은 프롬프트를 무한 루프 | Huntley의 원형 5줄 bash |
| **2세대 — Anthropic 공식 플러그인** | Claude Code 플러그인으로 정형화 | Anthropic 커뮤니티 |
| **3세대 — Codex /goal** | OpenAI 공식 기능으로 진입 | Codex CLI 0.128.0 |

5줄짜리 해커 스크립트가 OpenAI 공식 기능까지 진화한 길입니다. 이게 의미하는 바는 명확해요 — 자율 루프가 LLM 코딩의 **표준 패러다임**이 되고 있다는 거죠.

## 누적 진전 + 철학 (naive persistence)

{/* Source · DOC · Felipe Coury 트윗 인용 (Codex /goal 사용기) */}

Ralph Loop의 본질은 **단순 반복**이에요. AI가 똑똑한 게 아니라, 같은 작업을 안 멈추고 계속 시도하는 거예요. 작은 진척이라도 누적되면 하루 뒤에는 큰 결과가 나옵니다.

Felipe Coury가 한 줄로 요약했어요.

> "AI는 천재가 아니라 절대 멈추지 않는 노동자다."

이걸 **naive persistence**(순진한 끈질김)이라고 불러요. 사람은 5번 실패하면 그만두지만, AI는 5,000번도 시도합니다. 그 결과가 코드로 누적되면 사람의 5일치를 1시간에 만들어내요.

## Codex /goal — 5단계 라이프사이클

{/* Source · DOC · OpenAI Codex CLI 0.128.0 changelog (운영자 추출) */}

Codex의 `/goal`은 Ralph Loop을 정형화한 5단계 라이프사이클로 만들었어요.

```
1. Activation   ← /goal "..." 명령으로 활성화
2. Planning     ← AI가 작업을 잘게 쪼갬
3. Execution    ← 각 작업을 자율 실행
4. Verification ← 자체 검증 (테스트·빌드)
5. Soft Stop    ← 종료 조건 또는 사용자 중단
```

각 단계에서 사람이 끼어들 필요가 없어요. 단, 끼어들 수는 있게 설계됐습니다 — 그게 "Soft Stop"이에요.

## Soft Stop — 자율과 통제권의 균형

자율 루프의 가장 큰 우려는 "도망가면 어쩌나"예요. AI가 잘못된 방향으로 5,000번 반복하면 그게 다 토큰이고 비용이고 잘못된 코드입니다.

Codex /goal의 Soft Stop은 4중 안전장치를 깔았어요.

| 안전장치 | 무엇 |
| --- | --- |
| **Max iterations** | 기본 50회. 그 이상은 자동 중단 |
| **Token budget** | 초과 시 자동 중단 |
| **User interrupt** | Cmd-C로 즉시 중단 |
| **Self-doubt signal** | AI가 "방향이 맞는지 모르겠다" 신호 보내면 일시 정지 |

이 4중 안전장치가 깔리면 자율 루프를 안심하고 켜놓고 잘 수 있어요. 사람이 매 단계 옆에 있을 필요가 없습니다.

## 기존 LLM 코딩과의 비교

| 항목 | 기존 LLM 코딩 | Ralph Loop / /goal |
| --- | --- | --- |
| 사람 개입 | 매 단계 | 시작·종료만 |
| 진행 단위 | 메시지 1개 | 작업 N개 누적 |
| 실패 대응 | 사람이 다시 지시 | AI가 스스로 재시도 |
| 적합한 작업 | 짧은 1회성 | 긴 다단계 (테스트 추가, 마이그레이션) |
| 비용 통제 | 사람이 관리 | Soft Stop 자동 |

핵심은 "어느 게 좋다"가 아니라 **작업 모양에 따라 골라야 한다**는 거예요. 30분짜리 한 함수 수정에 Ralph Loop은 과해요. 하지만 200개 테스트 추가는 자율 루프가 압도적입니다.

## /goal 활성화 + 3중 격리

Codex에서 `/goal`을 켜는 흐름이에요.

```bash
# 1. 디렉토리 분리 (격리)
mkdir codex-experiment && cd codex-experiment
git init

# 2. 권한 설정 (allowlist만 허용)
codex config set permissions.allow "Read(**),Bash(npm run lint),Bash(npm run test)"

# 3. 목표 입력
codex
> /goal "src/utils의 모든 함수에 단위 테스트를 추가하라. npm run test가 통과해야 함."
```

3중 격리(독립 디렉토리 + git 분리 + 권한 허용 목록)가 깔리면 /goal이 폭주해도 본 프로젝트는 안전해요. [Lesson 29](/lessons/tool-permission-safeguards)의 권한 설계가 여기서 빛을 발합니다.

## 토큰 우려와 4중 안전장치

자율 루프의 두 번째 우려는 토큰 비용이에요. 50회 반복하면 단순 호출보다 50배 비싸 보이지만, 실제는 다릅니다.

| 토큰 측면 | 실제 |
| --- | --- |
| 입력 비용 | 컨텍스트가 누적되며 증가하지만 cache hit이 동작함 |
| 출력 비용 | 작업이 완료되면 더 이상 출력 없음 |
| 검증 비용 | 자체 검증이 사람 검토를 대체 |
| 실패 비용 | Soft Stop이 폭주를 막음 |

50배가 아니라 보통 5~10배 더 비싼 정도예요. 그 대신 사람의 5일을 1시간으로 줄여주면 비용 대비 가치가 압도적입니다.

## 그래서 오늘 만들 한 장

오늘 만들 건 **Ralph Loop / /goal 한 번 돌려본 기록**이에요.

| 단계 | 무엇 |
| --- | --- |
| 작업 선택 | 본인 프로젝트에서 30분 이상 걸릴 단순 반복 작업 1개 (예: 모든 함수에 의도 주석, 200개 라인 마이그레이션) |
| 격리 | 별도 디렉토리 + git 초기화 + 권한 허용 목록 |
| /goal 입력 | 명확한 종료 조건 포함 ("`npm run check`가 exit 0") |
| 모니터링 | 50회 또는 토큰 한도까지 관찰. Soft Stop 트리거 확인 |
| 비교 기록 | 사람이 직접 했을 때 예상 시간 vs Ralph Loop 실제 시간 + 토큰 비용 |

한 번 돌려보면 자율 루프의 감이 손에 잡혀요.
