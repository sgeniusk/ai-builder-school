## 정책이 없으면 각자가 기준이 된다

팀원 8명이 AI를 쓰는데 누가 어떻게 쓰는지 파악이 안 된다. 누군가는 고객 데이터를 ChatGPT에 통째로 넣고, 누군가는 AI 출력을 검토 없이 고객에게 보내고, 누군가는 AI가 쓴 내용을 본인 결과물로 제출한다.

'AI를 잘 써라'는 말만으로는 안 된다. **무엇을 맡기고 무엇을 검토할 것인지** 팀 기준이 있어야 한다.

---

## 3레이어 정책 구조

모든 걸 금지하거나 모든 걸 허용하는 극단을 피한다. 세 레이어로 나눈다.

### 금지 (Never)
팀 규모와 상관없이 해서는 안 되는 것.

- 고객의 개인정보(이름, 이메일, 연락처, 결제 정보)를 외부 AI 서비스에 입력
- 법적 문서, 계약서를 AI 출력 그대로 서명/발행
- 의료·금융 조언을 검토 없이 고객에게 전달
- AI 생성 내용을 본인 창작물로 표절

### 자유롭게 사용 (OK)
검토 없이 써도 되는 것.

- 내부 아이디어 브레인스토밍
- 초안 작성 (반드시 사람이 편집)
- 코드 리뷰 보조, 테스트 케이스 생성
- 개인 학습 및 정보 탐색

### 필수 검토 후 사용 (Review Required)
AI 결과물을 사용하되 사람이 반드시 검토해야 하는 것.

- 고객 응대 메시지 (발송 전 반드시 읽어볼 것)
- 마케팅 카피 (사실 확인 + 브랜드 톤 확인)
- 보고서·분석 결과 (수치와 인과관계 확인)
- 채용 관련 문서 (편향 가능성 확인)

## 핵심 이슈 4가지

### 개인정보 경계

```
원칙: 실명, 이메일, 전화번호, 주민번호, 결제 정보가 포함된 데이터는
      외부 AI 서비스(ChatGPT, Claude, Gemini 등)에 입력하지 않는다.

허용: 익명화된 데이터, 가상 데이터, 공개 정보
금지: 고객 DB 내용 직접 붙여넣기, CRM 데이터 그대로 입력
```

의심스러우면 데이터를 가상으로 바꾸거나 ID만 사용한다.

### AI 과의존 방지

```
원칙: AI는 초안 도구이지 결정 도구가 아니다.
      다음 유형의 작업은 AI 초안을 기반으로 하더라도
      최종 판단은 사람이 한다:
      - 고객과 직접 소통하는 모든 메시지
      - 숫자와 통계를 포함한 보고서
      - 채용, 평가, 인사 결정 관련 문서
```

### 저작권 기준

```
원칙: AI 생성 이미지, 코드, 글을 외부에 발행할 때는
      해당 AI 서비스의 저작권 정책을 확인한다.

주의: 특정 작가/작품의 스타일을 지정해 생성한 이미지는
      상업적 사용 전 법무팀 확인.
```

### AI 표시 의무

```
원칙: 다음 상황에서는 AI 사용 사실을 명시한다:
      - 블로그/SNS 등 외부 발행 콘텐츠
      - 제안서/보고서 (클라이언트에게 AI 보조 작성 사실 고지)

팀 내부 문서: 표시 의무 없음 (자유 사용)
```

## 1페이지 정책 문서의 톤

규정집이 아닌 **가이드**로 쓴다.

나쁜 톤:
> 제14조. 생성형 AI 도구의 활용 시 본 지침에 의거하여...

좋은 톤:
> AI를 쓸 때 우리가 함께 지키는 것. 모르는 게 있으면 물어보세요.

팀원이 읽고 싶어야 읽는다. 10분 안에 다 읽을 수 있는 분량이어야 한다.

---

## 온보딩 체크리스트

새 팀원이 합류할 때 이 체크리스트로 정책 이해를 확인한다.

```
[ ] AI 정책 문서를 읽었다
[ ] 금지 레이어(Never)의 내용을 말할 수 있다
[ ] 개인정보 경계가 어디인지 이해했다
[ ] '필수 검토' 레이어에서 검토를 건너뛰면 어떻게 되는지 안다
[ ] 정책이 불명확한 상황에서 누구에게 물어야 할지 안다
```

---

## 정책 업데이트 주기

AI 도구와 팀 상황은 빠르게 변한다. 정책도 살아있어야 한다.

- **분기 1회:** 금지/허용 레이어가 현재 도구에 맞는지 검토
- **인시던트 발생 시:** 해당 케이스를 반영해 즉시 업데이트
- **새 AI 도구 도입 시:** 도구 특성에 맞는 사용 기준 추가

버전 관리를 한다. `v1.0 → v1.1` 변경 이력과 이유를 기록한다.
