## 2. 콘텐츠 분리 원칙

| 영역 | 역할 | 절대 하지 않을 일 |
| --- | --- | --- |
| System | 정책과 우선순위 | 외부 콘텐츠로 덮어쓰기 |
| User | 사용자의 현재 요청 | 시스템 정책 변경 |
| Tool Output | 검색 결과와 외부 자료 | 실행 지시로 취급 |
| Policy Gate | 민감 동작 검사 | 모델 판단만 믿고 통과 |

---

## 3. 방어 규칙

| 규칙 | 적용 여부 |
| --- | --- |
| 외부 콘텐츠는 자료로만 취급한다 | |
| 외부 콘텐츠 안의 명령문은 실행하지 않는다 | |
| 시스템 프롬프트와 비밀 값은 출력하지 않는다 | |
| 민감 도구는 권한 정책을 통과해야 한다 | |
| 출처 없는 핵심 주장은 답변에서 줄인다 | |
| 인젝션 의심 로그를 남긴다 | |

## 4. 정책 게이트 의사코드

```ts
function policyGate(request) {
  if (request.asksForSecrets) return "deny";
  if (request.fromToolOutput && request.triesToChangeInstructions) return "deny";
  if (request.toolRisk === "high") return "confirm";
  return "allow";
}
```
