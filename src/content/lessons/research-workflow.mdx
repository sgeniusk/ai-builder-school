## "AI가 빠르게 찾아줬는데, 출처가 없어요"

상사가 묻습니다. "이 시장 올해 전망 어때?" AI에게 물어봅니다. 30초 만에 깔끔한 3문단 요약이 나와요. 좋아서 보고서에 붙이려는 순간 — "이 숫자 출처가 뭐야?" 답을 못 합니다. AI가 준 요약에 출처가 없거든요. 링크를 달라고 하면 그럴듯하지만 존재하지 않는 URL을 만들어 줍니다.

[긴 문서를 AI에게 먹이는 법](/lessons/feeding-long-documents)에서 "긴 문서를 어떻게 넣을지"를 배웠다면, 오늘은 **문서를 찾는 단계부터** 다룹니다. 질문 분해 → 검색 → 요약 → 근거 정리 → 보고. 이 다섯 단계를 파이프라인으로 만드는 게 오늘의 목표예요.

## 리서치가 보고서·회의록과 다른 점

{/* Source · DOC · Anthropic "Prompt engineering overview" — https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/overview */}

[보고서 초안 자동화](/lessons/automate-report-drafts)의 보고서, [회의록 → Action Item 파이프라인](/lessons/meeting-notes-pipeline)의 회의록은 **입력이 정해져 있었어요**. 데이터가 있고, 형식이 있고, 채우면 됩니다. 리서치는 다릅니다.

| 보고서/회의록 | 리서치 |
| --- | --- |
| 입력이 정형 (수치, 녹취) | 입력이 열림 (무엇을 찾아야 할지 정해야 함) |
| 형식이 있음 (템플릿) | 형식이 유동적 (질문에 따라 달라짐) |
| 사실 검증 범위가 좁음 | 사실 검증이 핵심 |

그래서 리서치에서는 **질문을 먼저 잘 쪼개는 것**이 전체 품질을 결정합니다.

## 질문 분해 — 큰 질문을 작은 질문으로

{/* Source · DOC · Anthropic "Prompt engineering overview" — https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/overview */}

"이 시장 올해 전망 어때?"는 너무 큰 질문이에요. AI에게 이대로 던지면 "평균적으로 그럴듯한 답"이 나옵니다. 대신 **하위 질문으로 쪼개면** 각 질문의 답이 구체적이고 검증 가능해져요.

```
큰 질문: "한국 B2B SaaS 시장의 2025년 전망은?"

하위 질문:
1. 2024년 한국 B2B SaaS 시장 규모는 얼마였는가? (수치 + 출처)
2. 주요 성장 드라이버 3가지는? (근거 있는 것만)
3. 주요 리스크 3가지는? (근거 있는 것만)
4. 경쟁사 3~5개의 최근 움직임은?
5. 글로벌 트렌드가 한국 시장에 미치는 영향은?
```

이 5개 하위 질문 각각에 AI를 호출하면, "큰 질문 한 번"보다 답의 밀도가 올라갑니다. 그리고 각 답에 **출처를 요구할 수 있어요**.

## 인용 강제 모드 — "출처 없으면 말하지 마"

{/* Source · DOC · Anthropic "Citations" — https://docs.anthropic.com/en/docs/build-with-claude/citations */}

리서치에서 가장 위험한 건 **출처 없는 주장**이에요. AI가 자신 있게 말하면 사람이 그냥 믿는 경우가 많습니다.

이걸 막는 가장 직접적인 방법은 **프롬프트에서 인용을 강제하는 것**이에요.

```
Constraints:
- 모든 주장에 출처(보고서명, 기관, 발행연도)를 반드시 붙이세요
- 출처를 찾을 수 없는 주장은 "출처 미확인"으로 표시하세요
- 절대로 URL을 만들어내지 마세요
- 확실하지 않은 수치는 "추정" 또는 "미확인"을 명시하세요
```

"출처를 찾을 수 없으면 말하지 마"보다 **"출처 미확인으로 표시해"**가 더 잘 먹힙니다. 전자는 AI가 아예 답을 안 하거나, 있는 출처를 억지로 갖다 붙이는 경향이 있어요. 후자는 정직한 표시를 유도합니다.

Anthropic의 Citations 기능을 활용하면 API 레벨에서 인용 근거를 문서 내 특정 위치에 연결할 수 있어요. 문서를 소스로 제공하고 모델이 답할 때 정확한 인용 위치를 반환받는 방식입니다.

## 근거 표 — 리서치의 중간 산출물

검색 결과를 바로 요약하면 안 돼요. 중간에 **근거 표**를 두세요. 이게 리서치 품질의 핵심이에요.

| 하위 질문 | 출처 | 핵심 인용 | 신뢰도 | 메모 |
| --- | --- | --- | --- | --- |
| 2024 시장 규모 | 과기부 보고서 (2024.06) | "국내 SaaS 시장 X조원" | 높음 | 정부 통계 |
| 성장 드라이버 1 | 가트너 보고서 (2024.09) | "클라우드 전환 가속" | 중간 | 글로벌 기준 |
| 리스크 1 | 업계 기사 (2024.11) | "보안 규제 강화" | 낮음 | 단일 기사 |

이 표가 있으면 두 가지가 가능해져요.

1. **약한 근거를 한눈에 파악** — 신뢰도 "낮음"이 많으면 추가 검색이 필요한 부분
2. **반박 근거 확인** — "이 주장과 반대되는 근거는 없는가?"를 의식적으로 추가

## 반박 근거 — 리서치의 숨은 벽

{/* Source · DOC · Anthropic "Prompt engineering overview" — https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/overview */}

AI 리서치에서 가장 빠지기 쉬운 함정이 **확증 편향**이에요. AI에게 "이 시장이 성장할 근거를 찾아줘"라고 하면, 성장 근거만 찾아줍니다. 반대 근거는 안 찾아요.

이걸 깨려면 의식적으로 반박 질문을 추가해야 해요.

```
추가 질문:
- 위 성장 드라이버에 대한 반박 근거가 있는가?
- 이 전망이 틀릴 수 있는 시나리오는?
- 동일 주제에 대해 부정적 전망을 내놓은 보고서가 있는가?
```

근거 표에 "찬성"과 "반박"을 나란히 놓으면 결론의 신뢰도가 올라갑니다.

## 그래서 오늘 만들 한 폴더

오늘 만들 건 `research-workflow/` 폴더예요. 리서치 주제 하나를 골라 이 파이프라인을 한 번 돌려 봅니다.

| 파일 | 역할 |
| --- | --- |
| `decompose.md` | 질문 분해 프롬프트 (큰 질문 → 하위 질문 5개) |
| `search-and-cite.md` | 인용 강제 모드 검색 프롬프트 |
| `evidence-table.md` | 근거 표 템플릿 (출처·인용·신뢰도·반박) |
| `synthesis.md` | 근거 표 → 최종 요약 프롬프트 |

이 네 파일이 리서치의 뼈대예요. 질문을 바꾸면 어떤 주제에든 적용됩니다.

| 단계 | 무엇 | 시간 |
| --- | --- | --- |
| 1. 분해 | 큰 질문 → 하위 질문 5개 | 5분 |
| 2. 검색 | 하위 질문별 인용 강제 검색 | 15분 |
| 3. 정리 | 근거 표 채우기 + 반박 근거 추가 | 10분 |
| 4. 종합 | 근거 표 → 최종 요약 + 신뢰도 명시 | 10분 |
| 5. 검수 | 사람이 "출처 미확인" 항목 직접 확인 | 10분 |
