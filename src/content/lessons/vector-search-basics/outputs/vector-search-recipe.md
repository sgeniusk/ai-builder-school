> AI Builder School · Lesson 6.3 출발 템플릿입니다.
> 청크와 임베딩을 검색 저장소에 넣고, 질문마다 Top-K 근거 후보를 가져오는 흐름을 설계합니다.

---

## 2. 인덱싱 흐름

```txt
1. 원문 문서를 불러옵니다.
2. 청킹 전략에 따라 content 조각을 만듭니다.
3. 각 청크에 메타데이터를 붙입니다.
4. 각 청크 content를 임베딩합니다.
5. 검색 저장소에 upsert합니다.
6. 실패한 청크 ID를 로그로 남깁니다.
```

---

## 3. 검색 함수 초안

```txt
Input.
- question
- topK
- filters

Steps.
1. question을 임베딩합니다.
2. filters로 검색 범위를 제한합니다.
3. 벡터 유사도 기준 Top-K 청크를 가져옵니다.
4. 필요하면 키워드 검색 결과와 합칩니다.
5. score와 source를 포함해 반환합니다.

Output.
- chunkId
- score
- content
- source
- metadata
```
