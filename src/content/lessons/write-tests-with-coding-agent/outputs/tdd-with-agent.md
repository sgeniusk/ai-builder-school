## 2. 실패 테스트 요청 프롬프트

```md
지금은 구현하지 말고 실패 테스트만 작성해 주세요.

규칙.
- Golden Path 1개와 Edge Case 2개 이상을 작성합니다.
- 각 테스트 이름은 사용자의 행동과 기대 결과가 보이게 씁니다.
- 각 테스트 위에 왜 필요한지 한 줄 주석을 남깁니다.
- 테스트 작성 후 구현하지 말고 멈춥니다.
- 실패 이유와 다음 구현 후보만 요약합니다.
```

## 4. 사람 리뷰

- [ ] 테스트 이름만 읽어도 요구사항이 보인다.
- [ ] Golden Path 가 기능의 중심을 잡는다.
- [ ] Edge Case 가 실제 위험을 반영한다.
- [ ] 구현 세부사항에 과하게 묶이지 않는다.
- [ ] 빠진 수용 조건이 없다.

---

## 5. Green 요청 프롬프트

```md
위 실패 테스트를 통과시키는 최소 구현만 해 주세요.

규칙.
- 계획에 없는 파일은 수정하지 않습니다.
- 테스트를 약하게 바꾸지 않습니다.
- 통과 후 어떤 변경을 했는지 요약합니다.
- 필요한 검증 명령을 실행하고 결과를 보고합니다.
```
