# 제약 사항 우선 해석 프롬프트

AI에게 A/B 테스트 결과를 해석시킬 때 낙관적 편향을 방어하는 프롬프트.

---

## 1단계 — 제약 사항 먼저 물어보기

```
아래 A/B 테스트 결과를 해석해줘.

[실험 데이터]
- A (기존): 전환율 <X.X%>, n=<N명>
- B (신규): 전환율 <X.X%>, n=<N명>
- p-value: <X.XXX>
- 테스트 기간: <N일>
- 테스트 변수: <무엇을 바꿨는지>

먼저 이 결과를 신뢰하기 어려운 이유나 제약 사항을 3~5가지 말해줘.
그 다음에 아래 세 가지 중 하나로 권고 사항을 말해줘:
- 배포 (신뢰할 수 있는 결과, 즉시 배포)
- 보류 (결과는 긍정적이지만 추가 검증 필요)
- 재실험 (신뢰하기 어려움, 더 엄밀한 설계로 재시도)
```

---

## 2단계 — 샘플 크기 계산

```
현재 기준선 전환율이 <X.X%>일 때,
최소 <Y%p> 차이(절대값)를 감지하려면
95% 신뢰수준, 80% 파워 기준으로 각 그룹에 몇 명이 필요한지 계산해줘.

현재 샘플 크기(<N명>)로 어느 정도 차이를 감지할 수 있는지도 알려줘.
```

---

## 3단계 — 실용적 유의미성 평가

```
이번 테스트에서 B가 A보다 <+X.Xp%> 높았어.
이 차이가 실제 비즈니스에 의미 있는지 판단해줘.

다음 맥락을 반영해줘:
- 월 트래픽: 약 <N>명
- 현재 전환율: <X.X%>
- 개발/운영 비용: <높음 / 낮음 / 모름>

연간 전환 건수 변화를 계산하고, 이 차이가 배포를 정당화할 만큼 큰지 의견을 줘.
```

---

## AI 응답 검토 포인트

AI의 해석을 받은 후 확인:

- [ ] 제약 사항을 3가지 이상 구체적으로 언급했는가
- [ ] 권고 사항(배포/보류/재실험)이 명확한가
- [ ] 통계적 유의미성과 실용적 유의미성을 구분했는가
- [ ] "유의미하다 = 배포해야 한다"로 단순화하지 않았는가

제약 사항 없이 "배포를 권장합니다"만 나오면 프롬프트 1단계를 다시 시도한다.
