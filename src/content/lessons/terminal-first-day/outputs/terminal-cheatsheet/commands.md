# 터미널 명령어 레퍼런스

이 학교 실습에서 쓰는 명령어만 모았다. 외우지 않아도 된다. 필요할 때 찾아본다.

---

## 위치와 이동

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `pwd` | 현재 위치 출력 | `pwd` |
| `cd 폴더명` | 폴더 이동 | `cd Documents` |
| `cd ..` | 한 단계 상위로 | `cd ..` |
| `cd ~` | 홈으로 이동 | `cd ~` |
| `cd -` | 직전 위치로 | `cd -` |

## 파일과 폴더 보기

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `ls` | 폴더 목록 | `ls` |
| `ls -la` | 숨김 파일 포함 상세 목록 | `ls -la` |
| `cat 파일명` | 파일 내용 보기 | `cat README.md` |

## 파일과 폴더 만들기

| 명령어 | 기능 | 예시 |
|--------|------|------|
| `mkdir 폴더명` | 폴더 생성 | `mkdir my-project` |
| `touch 파일명` | 빈 파일 생성 (Mac/Linux) | `touch .env` |
| `new-item 파일명` | 빈 파일 생성 (Windows) | `new-item .env` |

## VS Code 열기

| 명령어 | 기능 |
|--------|------|
| `code .` | 현재 폴더를 VS Code로 열기 |
| `code 파일명` | 특정 파일을 VS Code로 열기 |

## Node.js / npm

| 명령어 | 기능 |
|--------|------|
| `node --version` | Node.js 버전 확인 |
| `npm --version` | npm 버전 확인 |
| `npm install` | 패키지 설치 |
| `npm run dev` | 개발 서버 시작 |

## 자주 쓰는 단축키

| 단축키 | 기능 |
|--------|------|
| `Tab` | 자동완성 |
| `↑ ↓` | 명령어 히스토리 |
| `Ctrl+C` | 명령 중단 |
| `Ctrl+L` | 화면 지우기 |

## 흔한 에러와 해결

**`No such file or directory`**
→ 폴더/파일 이름이 틀렸거나 대소문자가 다른 것. `ls`로 이름 확인 후 재시도.

**`command not found`**
→ 해당 프로그램이 설치되지 않았거나 이름이 틀린 것. 설치 여부 확인.

**`Permission denied`**
→ 권한 부족. Mac은 `sudo` 앞에 붙이되, 이 학교 실습에서는 거의 필요 없음.

**해결 안 될 때**
→ 에러 메시지 전체를 AI에 붙여넣고 운영체제(Mac/Windows)와 상황을 설명한다.
