## 2. Citation-required 프롬프트 (복사용)

```
다음 작업을 해 주세요.

규칙:
- 답의 각 주장 뒤에 [출처]를 표기합니다.
- 출처를 확신하지 못하면 답하지 말고 "확인 불가"라고 적은 뒤
  그 이유를 한 줄로 설명합니다.
- 추측·일반화 표현(아마, 일반적으로, 대체로)은 금지입니다.

질문: <여기에 진짜 질문을 넣으세요>
```

## 3. Chain of Verification (CoVe) 4단 루프

```
4단계로 답해 주세요.

1단계 — 다음 질문에 답하세요: <질문>
2단계 — 위 답에서 사실 검증이 필요한 주장 3개를 골라
        각 주장을 검증할 수 있는 질문 3개를 만드세요.
3단계 — 그 3개 질문에 새로 답하세요.
4단계 — 1단계 답과 3단계 답이 일치하는지 확인하고,
        불일치가 있으면 수정된 최종 답을 제시하세요.
```

**언제 쓰나**: 다단계 추론, 인용 포함된 답, 단정적 어조가 의심스러울 때.

## 4. Confidence 점수 요청 (한 줄 추가)

```
각 주장 뒤에 신뢰도(0~100)와 그 점수의 근거 한 줄을 함께 적어 주세요.
신뢰도가 60 이하인 주장은 "검증 필요"로 표시해 주세요.
```

**점수 해석 가이드**:
- 90+ : 일반 상식 / 학습 데이터에 충분히 등장한 패턴
- 60–89 : 추정 가능하지만 근거 확인 필요
- 60 미만 : 거의 환각 위험. 답을 그대로 쓰지 말 것
