## 2. MCP 서버 구성 — Tools · Resources

도구를 "동작"과 "데이터"로 나눠 적는다.

### Tools (AI가 실행하는 동작)

| 이름 | 하는 일 | 입력 파라미터 | 반환값 |
| --- | --- | --- | --- |
| 예: `search_wiki` | 사내 위키 검색 | `query: string` | 검색 결과 목록 |
| | | | |

### Resources (AI가 읽는 데이터)

| URI | 내용 | 형식 |
| --- | --- | --- |
| 예: `wiki://config` | 위키 접속 설정 | JSON |
| | | |

## 4. 클라이언트 연결 · 검증

### 연결 설정 (stdio 예시)

```json
{
  "mcpServers": {
    "my-tool": {
      "command": "node",
      "args": ["/절대경로/my-mcp-server.js"]
    }
  }
}
```

### 검증 체크리스트

- [ ] 클라이언트가 서버를 인식하고 Tools 목록이 보이는가
- [ ] Resources 목록이 보이는가
- [ ] 도구를 한 번 호출해 정상 결과가 오는가
- [ ] 잘못된 입력을 줬을 때 에러가 깔끔히 반환되는가
- [ ] API 키 같은 비밀값이 응답·로그에 노출되지 않는가

## 5. Function Calling 대비 차이

| 항목 | Function Calling | MCP 서버 |
| --- | --- | --- |
| 도구가 사는 곳 | 한 앱 코드 안 | 별도 서버 프로세스 |
| 재사용 범위 | 그 앱에서만 | MCP 지원 클라이언트 전부 |
| 연결 방식 | 앱마다 새로 작성 | 표준 인터페이스 (USB-C) |
| 노출 단위 | 함수 | Tools + Resources |
