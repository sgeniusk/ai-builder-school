## 1. 항상 허용되는 작업

> 되돌리기 쉽고, 저장소를 망가뜨릴 가능성이 낮은 작업을 적어요.

- 파일 읽기와 구조 파악.
- 관련 파일 후보 목록 작성.
- 타입체크, 린트, 테스트 실행.
- 작은 문서 수정 제안.
- 변경 전 계획서 작성.

---

## 2. 사람 확인 후 허용되는 작업

> 실행 비용이 있거나 영향 범위가 넓은 작업은 여기 둡니다.

- 패키지 설치 또는 의존성 변경.
- 설정 파일 변경.
- 여러 폴더에 걸친 리팩터링.
- 데이터 마이그레이션 스크립트 작성.
- 외부 서비스 호출이 필요한 작업.

확인 문장 예시.

```md
이 작업은 의존성 변경을 포함합니다. 진행해도 될까요?
```

---

## 3. 항상 금지되는 작업

> 한 번 실행하면 회복 비용이 큰 작업을 적어요.

- `.env` 또는 비밀 파일 읽기와 수정.
- 사용자 승인 없는 commit, push, reset.
- 운영 데이터 삭제.
- 비밀 값 로그 출력.
- 요청 범위를 벗어난 대규모 구조 변경.

---

## 4. 비밀 관리 규칙

| 대상 | 규칙 |
| --- | --- |
| `.env` | 읽거나 수정하지 않음. |
| `.env.example` | 키 이름만 두고 실제 값은 비움. |
| 로그 | 토큰, 이메일, 고객 정보 출력 금지. |
| 프롬프트 | 실제 API 키를 붙여 넣지 않음. |

---

## 5. 작업 로그 형식

에이전트는 마지막 응답에 아래 항목을 남깁니다.

- 읽은 핵심 파일.
- 수정한 파일.
- 실행한 검증 명령.
- 실패하거나 건너뛴 검증.
- 사람 확인이 필요한 남은 결정.
