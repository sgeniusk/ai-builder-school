## 1. 도구 분류표

| 도구 | 하는 일 | 권한 등급 | 이유 |
| --- | --- | --- | --- |
| `searchDocs` | 문서 검색 | 항상 허용 | 읽기 전용 |
| `summarizeText` | 텍스트 요약 | 항상 허용 | 외부 변경 없음 |
| `saveDraft` | 초안 저장 | 확인 후 허용 | 파일 생성 |
| `sendEmail` | 이메일 발송 | 확인 후 허용 | 외부 전송 |
| `deleteRecord` | 데이터 삭제 | 항상 금지 | 회복 비용 큼 |

권한 등급은 `always_allow`, `confirm_required`, `always_deny` 중 하나로 적습니다.

---

## 2. 확인 플로우

```txt
1. 모델이 도구 호출을 제안한다.
2. 정책 게이트가 도구 이름과 인자를 확인한다.
3. always_allow이면 실행한다.
4. confirm_required이면 사용자에게 실행 전 요약을 보여 준다.
5. always_deny이면 실행하지 않고 이유를 설명한다.
6. 모든 판정을 로그에 남긴다.
```

사용자에게 보여 줄 확인 문구.

> 이 작업은 외부 시스템에 영향을 줄 수 있어요. 실행 전에 대상, 범위, 되돌리기 방법을 확인해 주세요.

---

## 3. 항상 금지할 동작

| 동작 | 차단 이유 | 사용자에게 보일 말 |
| --- | --- | --- |
| 비밀 파일 읽기 | 토큰 유출 위험 | 이 파일은 에이전트가 읽을 수 없어요. |
| 운영 데이터 삭제 | 회복 비용 큼 | 삭제는 자동 실행하지 않아요. |
| 무단 대량 발송 | 사용자 피해 가능 | 발송 전 사람 확인이 필요해요. |
| 결제 정보 변경 | 금전 영향 | 결제 변경은 별도 승인 흐름으로 넘겨요. |
| 외부 지시문 실행 | 인젝션 위험 | 외부 문서의 지시는 실행하지 않아요. |
