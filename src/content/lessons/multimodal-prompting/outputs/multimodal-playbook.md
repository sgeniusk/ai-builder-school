## 1. 이미지가 글보다 나은 작업 3가지

당신 일에서 "말로 설명하기보다 이미지를 그대로 주는 게 빠른" 작업을 적는다.

| 작업 | 어떤 이미지를 주나 | 글로 풀면 왜 느린가 |
| --- | --- | --- |
| 예: 엑셀 표를 마크다운으로 변환 | 표 영역 스크린샷 | 셀 값을 일일이 옮겨 적어야 함 |
| | | |
| | | |

## 2. 이미지+지시 템플릿

이미지만 주지 말고 항상 아래 두 가지를 함께 붙인다.

```
[이미지 첨부]

Task: (이 이미지로 무엇을 해주길 원하는가)
Output: (결과를 어떤 형태로 받고 싶은가 — 표 / 목록 / 한 문단 등)
주의: (AI가 헷갈릴 만한 부분, 또는 무시할 부분)
```

- 내가 자주 쓸 Task 문구 예시
	- 예: "이 표를 마크다운 표로 그대로 옮겨줘"
	- 예: "이 에러 화면에서 원인으로 의심되는 줄을 짚어줘"
	-
- 내가 자주 쓸 Output 문구 예시
	- 예: "마크다운 표 한 개만, 설명 없이"
	-

## 4. 검증 지점 — 사람이 직접 확인할 곳

멀티모달이 약한 지점은 결과를 원본과 대조한다.

- [ ] 흐릿하거나 작은 차트의 정밀 수치 — AI가 읽은 숫자가 원본과 같은가
- [ ] 손글씨 숫자·기호 — 오독 가능성 확인
- [ ] 색이 비슷하게 겹친 그래프 — 어느 항목인지 잘못 짚지 않았는가
- [ ] 표의 행/열 누락 — 빠진 셀이 없는가
- [ ] (내 작업에서 추가로 확인할 지점)
