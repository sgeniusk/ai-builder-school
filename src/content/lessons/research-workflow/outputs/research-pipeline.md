# 리서치 파이프라인

> 이 파일은 AI Builder School · Lesson 3.3 에서 제공되는 출발 템플릿입니다.
> Question Decomposition + Evidence Table + Confidence Floor 가 한 페이지에. 진짜 필요한 리서치 주제로 채워서 쓰세요.

## 0. 리서치 주제

```
큰 질문: ___
독자: ___ (예 — 본인 / 상사 / 클라이언트)
인용 목적: ___ (예 — 발표 자료 / 내부 의사결정 / 기획안)
Confidence Floor: 4 이상만 인용 (1~5 척도)
```

---

## 1. Question Decomposition

> 큰 질문을 *검증 가능한* 하위 질문 5 개로 쪼개라.
> 각 하위 질문 옆에 *검색 키워드 3 개* 도 함께.

| # | 하위 질문 | 검색 키워드 | 가능한 출처 종류 |
| --- | --- | --- | --- |
| 1 | | | (공시 / 뉴스 / 학술 / 업계 리포트 / 내부 자료) |
| 2 | | | |
| 3 | | | |
| 4 | | | |
| 5 | | | |

> AI 프롬프트, *"이 큰 질문을 검증 가능한 하위 5 개로 쪼개고, 각 하위에 검색 키워드 3 개 + 가능한 출처 종류를 표로 정리하라."*

---

## 2. Evidence Table

각 하위 질문에 대해 **표 한 줄 이상** 채움. Confidence 3 이하면 1 차 자료 재확인 또는 인용 포기.

| 하위 # | 주장 | 출처 (URL/문서) | 인용문 (원문 그대로) | 검증 일자 | Confidence (1~5) |
| --- | --- | --- | --- | --- | --- |
| 1 | | | | | |
| 1 | | | | | |
| 2 | | | | | |
| 3 | | | | | |
| 4 | | | | | |
| 5 | | | | | |

> Confidence 척도,
> - 5 — 원본 1 차 자료 (공시 / 공식 통계 / 학술 논문 본문). 의심 여지 없음.
> - 4 — 인용된 2 차 자료지만 *원본 인용이 확인됨*. 인용 OK.
> - 3 — 업계 리포트·뉴스. 추세는 OK, 수치는 한 번 더 확인.
> - 2 — 블로그·SNS 인용. 별도 검증 없이는 인용 불가.
> - 1 — AI 가 만든 출처. **인용 금지**.

---

## 3. 답 종합 — Evidence Table 만 컨텍스트로

```
시스템 프롬프트:
"아래 Evidence Table 만 사용해 큰 질문에 답하라.
표에 없는 주장은 절대 추가 금지.
각 주장 뒤에 [#하위번호] 로 출처 표시.
결론 마지막에 'Confidence 종합' 을 1~5 로 표기 (가장 낮은 항목 기준)."

큰 질문: ___
Evidence Table:
| ... |
```

## 4. 최종 보고서 초안 자리

```markdown
# (보고서 제목)

## 핵심 결론
> Confidence 종합: __/5
- 결론 1 [1, 3]
- 결론 2 [2]
- 결론 3 [4]

## 하위 질문별 발견
### 1. (하위 질문)
- 주장 ([출처])
- 주장 ([출처])

### 2. ...

## 미검증 / 후속 리서치
- (Confidence 3 이하라 인용 안 한 항목)
```

---

## 5. 트러스트 체크 — Origin 축 집중

| 점검 | 결과 |
| --- | --- |
| 모든 주장에 출처 번호가 있는가 | |
| 출처 URL 이 실제로 그 내용을 담는가 (스폿 체크 3 개) | |
| 인용문이 *원문 그대로* (또는 정확한 paraphrase) 인가 | |
| Confidence 4 이상만 보고서에 들어갔는가 | |
| 반박 근거 / 한계점 섹션이 있는가 | |

## 6. 저장 구조

```
    ├── summary.md       (3.~4.)

```
