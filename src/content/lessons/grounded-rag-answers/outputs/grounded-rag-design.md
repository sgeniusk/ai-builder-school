# Grounded RAG 설계서

> AI Builder School · Lesson 6.4 출발 템플릿입니다.
> 검색된 근거 안에서만 답하고, 모든 답에 확인 가능한 출처를 붙이는 RAG 응답 규칙을 설계합니다.

## 0. 내 RAG 앱 맥락

| 항목 | 내용 |
| --- | --- |
| 앱 이름 | |
| 답변할 문서 범위 | |
| 사용자가 가장 자주 묻는 질문 | |
| 잘못 답하면 위험한 주제 | |
| 출처를 보여 줄 방식 | 링크 / 파일명 / 페이지 / 스니펫 |

---

## 1. 응답 스키마

```ts
type GroundedRagAnswer = {
  answer: string;
  citations: Array<{
    chunkId: string;
    documentTitle: string;
    section?: string;
    snippet: string;
    source: string;
  }>;
  confidence: "high" | "medium" | "low";
  missingEvidence?: string[];
};
```

---

## 2. 답변 규칙

| 규칙 | 결정 |
| --- | --- |
| 검색된 청크 안에 있는 내용만 답한다 | 예 |
| 모든 핵심 주장에 citation을 붙인다 | 예 |
| citations가 0개이면 답하지 않는다 | 예 |
| 검색 결과가 충돌하면 충돌을 말한다 | 예 |
| 문서에 없는 추정은 하지 않는다 | 예 |

---

## 3. 거부 응답 정책

사용자에게 보일 말.

> 현재 연결된 문서에서는 이 질문의 근거를 찾지 못했어요. 문서 범위를 넓히거나 질문을 더 구체적으로 바꿔 주세요.

거부해야 하는 상황.

| 상황 | 처리 |
| --- | --- |
| Top-K 결과가 모두 낮은 유사도 | 거부 |
| 검색된 청크에 답의 핵심 근거가 없음 | 거부 |
| 사용자가 볼 수 없는 문서가 필요함 | 권한 안내 |
| 근거끼리 서로 충돌함 | 충돌 사실을 설명하고 확정 답변 보류 |

---

## 4. 출처 표시 UI

| UI 요소 | 표시할 내용 |
| --- | --- |
| 답변 본문 | 짧은 답 + 필요한 조건 |
| 출처 번호 | 답 문장 옆 `[1]`, `[2]` |
| 출처 패널 | 문서 제목, 섹션, 스니펫, 링크 |
| 신뢰도 표시 | high / medium / low |
| 근거 부족 알림 | missingEvidence 요약 |

## 5. 수동 검증 질문

| 질문 | 기대 동작 | 실제 답변 | 출처 일치 여부 |
| --- | --- | --- | --- |
| 문서에 답이 있는 질문 | 답변 + citation | | |
| 문서에 없는 질문 | 거부 응답 | | |
| 비슷하지만 다른 정책 질문 | 정확한 문서만 인용 | | |
| 오래된 문서와 새 문서가 충돌하는 질문 | 충돌 표시 | | |
| 권한이 필요한 질문 | 권한 안내 | | |
