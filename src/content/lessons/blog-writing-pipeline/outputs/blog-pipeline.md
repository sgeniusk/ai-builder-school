# 블로그 글쓰기 파이프라인

> 이 파일은 AI Builder School · Lesson 3.4 에서 제공되는 출발 템플릿입니다.
> Style Capsule + 4-pass 편집 루틴이 한 페이지에. 발행 예정 글 한 편으로 끝까지 굴려보세요.

## 1. Style Capsule — 내 톤의 한 페이지

```yaml
opening_patterns:
  - "질문으로 시작"
  - "장면 묘사 (시간·장소·인물 한 줄)"
  - "작은 일화"
  - "_(추가)_"
sentence_length:
  avg: 짧음 (15~25자)
  long_sentence_freq: "한 단락에 1 개"
paragraph_length:
  avg_lines: 2~4
  one_sentence_paragraphs_ok: true
emoticon_use: "거의 없음"
code_block_use: "코드일 때만"
table_use: "비교 4 개 이상일 때"
ending_patterns:
  - "질문 던지기 (독자에게)"
  - "다음 글 / 다음 행동 예고"
  - "한 줄 요약"
forbidden_words:
  - "확실히"
  - "분명히"
  - "정말로"
  - "_그러므로_"
forbidden_clichés:
  - "한 번 쯤은"
  - "결국"
  - "마치 ~처럼"
voice_anchors:
  - "겸손하지만 단정적"
  - "구체 수치 인용"
  - "독자에게 말 거는 어조"
```

---

## 2. 샘플 3 편 (Voice Preservation)

> 자신 있게 *"내 글"* 이라 부를 수 있는 과거 글 3 편. 시스템 프롬프트에 *원문 그대로* 첨부.

```
[샘플 1 — 1,200자] (제목과 본문 그대로)
...

[샘플 2 — 1,500자]
...

[샘플 3 — 900자]
...
```

---

## 3. 시스템 프롬프트

````
너는 위 3 개 샘플 글의 톤을 따라 새 글을 쓰는 도우미다.

분석할 것:
- 어휘 선택 (자주 쓰는 단어 / 회피하는 단어)
- 문장 호흡 (길이, 단락 길이)
- 시작·끝 패턴
- 비유·예시 사용 패턴
- 강조·이모티콘 사용 패턴

규칙:
1. 새 주제에 대해 *그 톤* 그대로 작성.
2. Style Capsule 의 forbidden_words / forbidden_clichés 사용 금지.
3. 출력은 마크다운. 헤딩·문단 구분 자유.

새 주제: ___
목표 길이: ___
독자: ___
````

## 4. 4-pass 편집 루틴

| Pass | 작업 | 누가 | 시간 | 통과 기준 |
| --- | --- | --- | --- | --- |
| **1. Draft** | 초안 생성 | AI | 5분 | 길이만 OK |
| **2. Logic** | "흐름 끊긴 곳·인과관계 약한 곳 5개 짚어라" → 수정 | AI | 5분 | 논리 끊김 0 |
| **3. Tone** | 각 문단 첫 문장 소리내 읽기 → *"내 말투인가?"* 자가 검수. 아니면 다시 쓰기 | **사람 강하게** | 20~40분 | 50% 이상 사람이 손댐 |
| **4. Grammar** | 오타·맞춤법·표기 통일·인용부호 일관성 | AI | 3분 | 0 에러 |

> **Pass 3 가 핵심**. AI 의 매끄러움을 뚫고 *나의 거친 진심* 이 들어가는 자리.

---

## 5. 자기 점검 — Pass 3 검수 질문

각 단락 첫 문장에 답해보세요.

- 이 문장을 친구한테 *"이거 내가 쓴 거야"* 라고 보여줄 때 부끄러운가?
- 이 문장이 *내 글의 첫 문장* 이어도 부끄럽지 않은가?
- 이 단락이 *AI 가 쓴 거 같다* 는 신호 (지나친 매끄러움, 클리셰, 정보의 평균값) 가 있는가?

부끄러우면 다시.

## 6. 발행 직전 트러스트 체크 (Lesson 1.4)

| 축 | 점검 |
| --- | --- |
| Fact | 수치·이름·날짜·인용이 원본과 일치 |
| Origin | 외부 인용엔 출처 링크 / 자체 경험엔 *내 경험* 인지 |
| Scope | 단정적 일반화 ("모든 ~", "항상 ~") 정당한가 |
| Confidence | 강한 주장에 비례하는 근거가 있는가 |
| Bias | 한쪽 관점만 다루지 않았는가 |
