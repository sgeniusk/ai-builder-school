## 자동화가 매일 다른 이유로 깨지고 있어요

회의록 → Action Item JSON → Notion 자동 등록. 멋지게 짠 파이프라인인데, 일주일에 한두 번씩 깨집니다. 들여다보면 매번 원인이 달라요.

- 어제는 AI 가 `due_date` 필드를 *"2026년 6월 20일"* 한국어로 적어서 파서가 던졌습니다.
- 오늘은 `assignee` 가 배열일 때 단일 문자열로 답해서 깨졌어요.
- 그저께는 답 앞에 `좋습니다, 결과는 다음과 같습니다.` 같은 인삿말이 붙어 JSON 이 아니게 됐죠.

문제는 *모델이 매번 다른 실수를 한다는 것* 이 아니라 **출력 형식을 강제할 도구가 없다는 것** 입니다. 자동화가 시작되는 지점부터 형식의 *계약* 이 있어야 해요.

## 4축 다음에 왜 이 레슨인가

[Lesson 2.1](/lessons/structure-of-good-prompts) 의 4축 중 가장 자동화에 직결되는 게 *Output* 축이에요. 사람이 읽으면 "표든 글이든" 적당히 넘어가지만, 다른 시스템이 받으면 **한 글자만 어긋나도 전체 파이프라인이 멈춥니다**.

오늘은 그 *Output* 축을 한 단계 강하게 — JSON 스키마와 재시도 루프로 잠가요. 이 한 가지만 익혀도 *수동 → 자동화* 전환 비용이 크게 줄어듭니다.

## 핵심 개념 — Schema First / Retry Loop / Soft-Strict

### Schema First — *예시보다 스키마가 먼저*

JSON 스키마 또는 *예시 + 필드 설명* 을 프롬프트 안에 직접 넣어요. "이런 식으로 답해줘" 가 아니라 *"이 스키마 외 형식은 거절한다"* 의 톤.

```json
{
  "action_items": [
    {
      "assignee": "string (한국어 이름)",
      "task": "string (50자 이내)",
      "due_date": "string (YYYY-MM-DD)",
      "priority": "high | medium | low"
    }
  ]
}
```

이 스키마를 프롬프트에 넣고 *"오직 이 JSON 만 출력하라. 인삿말 금지. 마크다운 코드블록 금지."* 까지 명시.

### Retry on Parse Error — *실패해도 자동 회복*

실서비스에선 한 번에 100% 통과는 어렵습니다. 그래서 *재시도 루프* 가 필수.

```
try:
  result = JSON.parse(ai_output)
catch ParseError:
  retry with prompt + "직전 답이 JSON 으로 파싱되지 않았다. 수정된 JSON 만 반환하라."
```

3 회 까지 재시도하고도 실패하면 사람에게 에스컬레이트. 이게 자동화 시스템의 기본 안전망이에요.

### Soft-Strict — *유연성과 엄격성의 균형*

너무 엄격한 스키마는 모델의 자연스러운 추론을 가두기도 합니다. *핵심 필드는 strict, 부가 정보는 optional* 이 균형점.

| 축 | strict 권장 | optional 권장 |
| --- | --- | --- |
| 자동화 키 (담당자, 기한) | ✓ | |
| 분류 (priority, category) | ✓ (enum) | |
| 설명 / 근거 / 노트 | | ✓ |
| 메타 (생성 시간, 출처) | | ✓ |

핵심만 잠그고 나머지는 모델이 채우게 두면 *실패율 ↓ 정보량 ↑*.

## 그래서 오늘 할 일

매일 흘러가는 데이터 흐름 하나를 골라 *구조화 출력 미니 파이프라인* 으로 묶어요. 권장,

- 회의록 → Action Item JSON
- 고객 문의 → 분류·우선순위 JSON
- 월간 보고서 → 핵심 KPI JSON

작업 흐름,

1. 필요한 필드 5~8개 를 종이에 적습니다 (5분).
2. JSON 스키마 + 한 줄 예시를 작성합니다 (10분).
3. 이를 프롬프트에 직접 임베드합니다 (5분).
4. Node.js 또는 TypeScript 로 *호출 → JSON.parse → 실패 시 재시도* 루프를 짭니다 (15분).
5. 같은 입력으로 5회 호출해 100% 파싱 성공인지 확인 (5분).

산출물은 [`structured-output-example.ts`](/lessons/enforcing-output-format) — 스키마 + 호출 + 재시도가 50줄 이내로 들어 있어요. 이걸 자기 도메인으로 복제·수정.
