## 1. 작업 흐름 도식 — 시점별 검증

각 시점에 어떤 검증을 자동으로 걸지 정한다.

```
[작업 전] ──→ [작업 중] ──→ [작업 후] ──→ [커밋 전] ──→ 커밋 완료
   │                          │              │
 환경 점검                  결과 검증       차단형 게이트
```

| 시점 | 거는 검증 | 차단 여부 |
| --- | --- | --- |
| 작업 전 | 예: 의존성 설치 확인, 브랜치 확인 | [ ] |
| 작업 후 | 예: lint, 변경 파일 포맷 검사 | [ ] |
| 커밋 전 | 예: lint + typecheck + test | [ ] 차단 |
| | | [ ] |

## 2. hook 설정 (차단형)

실패 시 종료 코드를 0이 아닌 값으로 돌려 진행을 막는다.

- hook이 실행되는 시점:
- 실행 명령:
```bash
# 예시 — 하나라도 실패하면 exit 1 → 커밋 중단
npm run lint && npm run typecheck && npm test || exit 1
```
- [ ] 경고만 띄우는 게 아니라 실제로 진행이 막히는지 확인했다
- [ ] 통과했을 때는 조용히 넘어가는지 확인했다

## 4. 자가 수정 루프

실패 출력을 에이전트에 돌려 스스로 고치게 하는 절차.

1. **수집** — hook이 막은 출력(에러 메시지·스택)을 그대로 복사한다
2. **전달** — 그 출력을 에이전트에게 입력으로 준다 ("이 게이트가 막혔어요, 고쳐 주세요")
3. **재실행** — 에이전트가 고친 뒤 같은 hook을 다시 돌린다
4. **반복** — 통과할 때까지 1~3을 반복한다 (무한 루프 방지: 최대 N회)
