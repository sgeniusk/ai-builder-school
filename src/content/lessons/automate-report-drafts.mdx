## 매주 같은 형식, 다른 내용 — 그 반복이 시간을 다 먹어요

월요일 오전, 주간 보고서를 또 씁니다. 항상 같은 5 섹션 — 핵심 지표 / 진행 / 막힌 곳 / 다음 주 / 위험. 내용만 매주 바뀌고, 구조는 1년째 같은 모양. 그런데 매번 빈 페이지 앞에서 30분을 까먹어요. *"이번 주는 어떻게 시작하지"* 의 망설임이 가장 비싸요.

문제는 **창의가 부족한 게 아니라 부팅 비용이 큰** 거예요. 이미 다 알고 있는 구조를 매번 머리로 다시 호출하는 게 비효율. 오늘은 그 부팅을 AI 에게 넘기고, 내 시간은 *분석·판단* 에 씁니다.

## Phase 2 다음에 왜 이 레슨인가

[Lesson 2.1](/lessons/structure-of-good-prompts) 4축 + [Lesson 2.2](/lessons/enforcing-output-format) 스키마 + [Lesson 2.4](/lessons/build-personal-prompt-library) 라이브러리. 세 가지가 손에 들어왔으면 이제 *실제 일과* 에 박을 차례예요. 보고서는 가장 자주, 가장 정형화된 작업이라 자동화 ROI 가 가장 높습니다.

이 레슨은 **Phase 3 (AI Work OS) 의 출발점** 이에요. 매일 같은 자리에서 호출되는 *자동화 한 가지* 가 자리 잡으면 다음 자동화는 자연스럽게 따라옵니다.

## 핵심 개념 — Template Grounding / Draft-Review-Final / Section Schema

### Template Grounding — *공식 템플릿을 컨텍스트로 박는다*

AI 가 *내 회사의* 보고서 구조를 처음부터 추측하면 매번 일탈해요. *과거 보고서 3 개를 본문에 함께 넣고* 그 구조를 따르라고 명시하면 **공식 톤·섹션·길이가 한 번에 잡힙니다.**

```
시스템 프롬프트:
"아래는 우리 팀의 지난 3주 보고서다. 같은 구조·톤·길이로 이번 주 보고서
초안을 작성하라. 섹션 순서·헤딩 레벨·표 사용 패턴을 그대로 유지하라."

[보고서 W-3 원문]
[보고서 W-2 원문]
[보고서 W-1 원문]

이번 주 입력:
- 핵심 지표: ...
- 진행: ...
```

이게 *Few-shot grounding*. 모델이 매번 *내 팀* 의 스타일을 다시 학습.

### Draft → Review → Final — *3 단 루프*

한 번에 완성된 답을 받으려 하면 어디서 어떻게 틀렸는지 보이지 않아요. 3 단으로 쪼개면 각 단계마다 *어디를 고칠지* 가 명확해집니다.

| 단계 | 작업 | 누가 | 시간 |
| --- | --- | --- | --- |
| **Draft** | 입력만 채워서 초안 생성 | AI | 30초 |
| **Review** | AI 가 자기 답을 5축 (사실·범위·톤·길이·구조) 으로 자기 점검 | AI | 30초 |
| **Final** | 사람이 수치·고유명사·뉘앙스 검수 | 사람 | 5분 |

[Lesson 1.4](/lessons/checks-before-trusting-ai-output) 의 트러스트 체크가 **Final 단계와 정확히 같은 일** 이에요. 도구가 자연스럽게 연결됨.

### Section Schema — *섹션 누락 방지*

스키마를 JSON 으로 박아두면 모델이 한 섹션을 빠뜨리지 않아요. [Lesson 2.2](/lessons/enforcing-output-format) 의 강제 출력 패턴이 여기서 한 번 더 등장합니다.

## 그래서 오늘 할 일

매주 / 매월 *같은 사람에게 같은 형식* 으로 보내는 보고서 1 개를 골라 파이프라인을 만들어요. 권장 후보,

- 주간 / 월간 진행 보고
- 프로젝트 상태 업데이트
- 영업 / 마케팅 KPI 리포트

작업 흐름,

1. **과거 3 개 수집** — 가장 *완성도 높았던* 보고서 3 편 (10분)
2. **공통 구조 추출** — AI 에게 *"섹션·헤딩·표 패턴을 표로 정리하라"* (5분)
3. **입력 슬롯 정의** — 매주 바뀌는 데이터 필드 (예 — `metrics`, `blockers`, `next_priorities`) 5~8개 (10분)
4. **Few-shot 프롬프트 작성** — 과거 3 편 + 새 입력 슬롯 + 출력 스키마 (15분)
5. **첫 실행 + 5분 트러스트 체크 적용** — Lesson 1.4 체크리스트 (10분)
6. **저장** — `prompt-library/reports/<name>.md` 에 Front Matter 와 함께 (Lesson 2.4 의 위치) (5분)

산출물 [`report-pipeline.md`](/lessons/automate-report-drafts) — Template Grounding 슬롯 + 3 단 루프 체크 + 저장 약속이 한 페이지에 들어 있어요.

## 마무리 — 매주 30분이 살아납니다

첫 보고서 자동화 한 번이 자리 잡으면, 두 번째는 패턴을 복사. 세 번째부터는 *내 일* 의 모양이 *프롬프트 라이브러리의 모양* 으로 그대로 옮겨와요. [Lesson 3.2](/lessons/meeting-notes-pipeline) 에서는 *회의록* 자동화, [Lesson 3.3](/lessons/research-workflow) 에서는 *리서치* 자동화로 이어집니다.

오늘 만든 하나가 한 달 안에 5~7 개로 확장되면, AI Work OS 의 1 차 윤곽이 잡힙니다.
