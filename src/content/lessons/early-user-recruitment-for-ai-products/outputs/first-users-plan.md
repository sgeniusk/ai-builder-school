# 첫 사용자 5명 모집 계획

> 이 파일은 지음 · Stage 8 "첫 사용자 찾기" 에서 제공되는 출발 템플릿입니다.
> 당신 제품에 맞춰 빈칸을 채우세요. 예시 줄은 지우고 써도 됩니다.

## 1. 내 제품이 푸는 문제

한 문장으로. 누가, 어떤 상황에서, 무엇이 불편한가.

- 문제 한 줄:
	- 예: "혼자 일하는 디자이너가 견적서를 매번 수작업으로 만드느라 30분씩 쓴다."
- 이 문제를 겪는 사람의 한 줄 묘사:
	- 예: "프리랜서 / 1인 스튜디오 디자이너"

## 2. 사용자가 있는 곳 5곳 (워터링홀)

손가락으로 가리킬 수 있을 만큼 구체적으로. "개발자 커뮤니티" (X) → "OO 디스코드 #질문 채널" (O).

| # | 장소 (구체적으로) | 종류 | 내가 들어갈 방법 |
| --- | --- | --- | --- |
| 1 | 예: r/freelance 서브레딧 | 서브레딧 | 가입 후 질문글 관찰 |
| 2 | | | |
| 3 | | | |
| 4 | | | |
| 5 | | | |

## 3. 1:1 권유 메시지

영업 말투 금지. "당신 문제를 풀 것 같다"는 톤으로.

- 보낼 메시지 초안:
	```
	안녕하세요, OO 채널에서 ___ 때문에 고민하신다는 글을 봤어요.
	제가 그 문제를 풀려고 ___ 를 만들었는데, 한번 써보고 솔직한 느낌을 들려주실 수 있을까요?
	무료고, 5분이면 됩니다.
	```
- 메시지를 보낼 후보 3명 (이름/핸들 + 어디서 봤는지):
	1.
	2.
	3.
