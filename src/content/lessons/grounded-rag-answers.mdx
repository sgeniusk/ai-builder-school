## "답은 그럴듯한데 어디서 나온 건지 모르겠어요"

벡터 검색까지 붙인 RAG가 돌아갑니다. "우리 회사 환불 정책이 뭐야?"라고 물으면 깔끔한 답이 나와요. 그런데 고객 지원팀장이 묻습니다 — "이 답이 어느 문서에서 나온 거야? 고객한테 이걸 보내도 되는 거야?" 출처가 안 보이니 **답을 믿을 수 없고, 믿을 수 없으니 쓸 수 없습니다**.

더 위험한 건, 검색된 청크에 없는 정보를 모델이 **지어낸** 경우예요. [Lesson 03](/lessons/hallucination-verification)에서 다룬 환각이 RAG 안에서도 일어납니다. 검색 결과를 줬는데도 모델이 "가끔 창작"을 합니다.

오늘은 이 두 문제를 한 번에 잡습니다 — **답에 반드시 출처를 달고, 근거 없으면 "모른다"고 답하게 만드는** 패턴이에요.

## RAG 환각 — 검색이 있어도 생기는 이유

{/* Source · PAPER · Lewis et al. (2020) "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks" — https://arxiv.org/abs/2005.11401 */}

RAG가 환각을 완전히 없앨 거라 기대하지만, 현실은 다릅니다. 세 가지 경로로 환각이 들어와요.

| 경로 | 설명 | 예시 |
| --- | --- | --- |
| 검색 실패 | 관련 청크가 Top-K에 없음 | 환불 정책이 6번째 청크에 있는데 K=5 |
| 근거 무시 | 청크를 줬지만 모델이 자기 지식으로 답함 | 청크엔 "7일 이내"인데 "30일"이라고 답 |
| 부분 근거 | 청크의 일부만 쓰고 나머지를 창작 | "7일 이내 환불 가능"에 "배송비 별도 부담" 추가 (청크에 없음) |

세 경로 모두 **인용(citation)이 없으면 감지가 불가능**해요. 답만 보면 그럴듯하니까요. 그래서 인용을 강제하는 겁니다.

## 인용 강제 시스템 프롬프트 — 핵심 패턴

{/* Source · DOC · Anthropic "Citations" — https://docs.anthropic.com/en/docs/build-with-claude/citations */}

[Lesson 06](/lessons/enforcing-output-format)의 3단 방어선을 RAG에 적용합니다. 1선(프롬프트)에서 인용 규칙을 명시하는 게 출발이에요.

```
System:
당신은 사내 문서 기반 Q&A 시스템입니다.

규칙:
1. 아래 [검색 결과]에서 근거를 찾아 답하세요.
2. 모든 문장에 출처를 [Source: 청크ID] 형태로 붙이세요.
3. 검색 결과에 근거가 없는 질문에는 반드시 다음과 같이 답하세요:
   "제공된 문서에서 해당 정보를 찾을 수 없습니다."
4. 추측하지 마세요. 근거가 부분적이면 "부분적으로 확인됨"이라 표시하세요.

[검색 결과]
[chunk-01] 환불은 구매 후 7일 이내 가능합니다.
[chunk-02] 환불 시 원래 결제 수단으로 환급됩니다.
[chunk-03] 개봉 상품은 환불 불가입니다.
```

이 패턴의 핵심은 **3번 규칙**이에요. "모른다"를 허용해야 환각이 줄어듭니다. 모델에게 "항상 답해"라고 하면, 근거가 없어도 지어내서라도 답합니다.

## Anthropic Citations API — API 레벨 인용

{/* Source · DOC · Anthropic "Citations" — https://docs.anthropic.com/en/docs/build-with-claude/citations */}

프롬프트로 인용을 부탁하는 건 1선이에요. Anthropic은 **API 레벨에서 인용을 강제**하는 Citations 기능을 제공합니다.

문서 청크를 `document` 소스로 전달하면, 모델이 답변할 때 어느 문서의 어느 부분을 참조했는지를 **구조화된 형태**로 반환해요.

```typescript
// 개념 예시 — 실제 파라미터는 공식 문서 참조
const response = await anthropic.messages.create({
  model: "claude-sonnet-4-20250514",
  messages: [{ role: "user", content: "환불 정책이 뭐야?" }],
  // documents 파라미터로 청크 전달
  // → 응답에 citations 포함
});

// 응답 예시:
// "환불은 구매 후 7일 이내 가능합니다 [1].
//  원래 결제 수단으로 환급됩니다 [2]."
// citations: [{source: "chunk-01", ...}, {source: "chunk-02", ...}]
```

1선(프롬프트)과 달리 **모델이 인용을 빼먹을 수 없어요**. API가 구조적으로 붙여주니까요. 이게 [Lesson 06](/lessons/enforcing-output-format)의 2선(스키마 강제)에 해당합니다.

## "근거 없으면 모른다" — Abstain Policy

{/* Note: Abstain Policy는 이 학교 운영자의 정리이며, 외부 표준 명칭은 아닙니다. */}

인용을 강제해도 한 가지가 더 필요해요. 검색 결과에 답이 없을 때 **"모른다"고 답하게 만드는 정책**입니다.

이걸 빼면 모델은 이렇게 합니다:
- 검색 결과에 없는 정보를 자기 훈련 데이터에서 끌어와 답함
- 인용 형식은 지키되 **가짜 청크 ID**를 만들어 붙임

Abstain Policy의 구현은 간단해요.

```
거절 조건:
- 검색 결과의 유사도가 모두 0.5 미만일 때
- 검색 결과에서 질문 주제와 관련된 키워드가 하나도 없을 때

거절 응답 형식:
"제공된 문서에서 해당 정보를 찾을 수 없습니다.
관련 문서를 추가하거나 다른 표현으로 질문해 주세요."
```

이 정책은 **프롬프트 + 코드 양쪽**에서 걸어야 합니다. 프롬프트에만 쓰면 모델이 무시할 수 있어요. 검색 결과의 유사도 점수를 코드에서 먼저 확인하고, 임계값 이하면 LLM을 아예 호출하지 않는 게 가장 확실해요.

## 인용 검증 — 답이 정말 근거와 일치하는가

인용이 달려 있어도 **답 내용과 청크 내용이 다를 수 있어요**. 모델이 청크를 잘못 해석하거나, 여러 청크를 섞으면서 없는 내용을 만들어낼 수 있습니다.

검증 방법 두 가지:

**1. 사후 검증 (Post-generation check)**
답변 생성 후, 각 인용된 청크와 해당 문장을 **별도 LLM 호출로 비교**합니다. "이 문장이 이 청크에서 지지되는가?" Yes/No.

**2. 사용자 검증 UX**
인용 옆에 [원문 보기] 버튼을 넣어 사용자가 직접 확인할 수 있게 합니다. 가장 비용이 낮고 가장 확실한 방법이에요.

| 방법 | 비용 | 정확도 | 지연 |
| --- | --- | --- | --- |
| 사후 LLM 검증 | LLM 호출 추가 | 높음 | 답변 시간 증가 |
| 사용자 검증 UX | 개발비만 | 가장 높음 | 사용자에게 달림 |
| 둘 다 | 높음 | 최고 | 답변 시간 증가 |

고위험 도메인(의료, 법률, 금융)이라면 둘 다 거세요. 사내 FAQ 수준이라면 사용자 검증 UX만으로도 충분합니다.

## End-to-End 파이프라인 — L23~L26의 총 조립

오늘까지 4레슨이 모이면 RAG 파이프라인 하나가 완성됩니다.

```
[L24 청킹] → [L23 임베딩] → [L25 벡터 DB 저장]
                                     ↓
                              [L25 벡터 검색]
                                     ↓
                         [L26 인용 강제 + Abstain]
                                     ↓
                              근거 박힌 답변
```

| 단계 | 레슨 | 하는 일 |
| --- | --- | --- |
| 1. 청킹 | L24 | 문서를 적절한 단위로 자르기 |
| 2. 임베딩 | L23 | 각 청크를 벡터로 변환 |
| 3. 저장 | L25 | 벡터 DB에 인덱싱 |
| 4. 검색 | L25 | 쿼리와 가장 유사한 Top-K 가져오기 |
| 5. 답변 | L26 (오늘) | 인용 붙여 답하거나, 근거 없으면 거절 |

## 그래서 오늘 만들 폴더

오늘 만들 건 `rag-pipeline/` 폴더예요 — L25의 검색 파이프라인 위에 LLM 답변 + 인용을 얹은 end-to-end 데모입니다.

| 파일 | 역할 |
| --- | --- |
| `chunk.py` | 문서 청킹 (L24에서 만든 것 재사용) |
| `embed.py` | 청크 임베딩 + 벡터 DB 저장 (L25 재사용) |
| `search.py` | Top-K 검색 (L25 재사용) |
| `answer.py` | 검색 결과 + 시스템 프롬프트 → 인용 포함 답변 |
| `abstain.py` | 유사도 임계값 체크 → 거절 로직 |
| `eval.md` | 테스트 질문 10개 + 답변 + 인용 정확도 |

이 폴더가 Stage 6의 최종 산출물이에요. 문서를 넣으면 근거 있는 답이 나오고, 근거가 없으면 "모른다"고 합니다.
