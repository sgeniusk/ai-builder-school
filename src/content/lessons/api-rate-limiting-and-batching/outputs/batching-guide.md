# 배치 프롬프팅 가이드

> 이 파일은 AI Builder School · Stage 2 "배치 프롬프팅과 비용 최적화" 에서 제공되는 출발 템플릿입니다.
> 당신이 실제로 반복하는 작업에 맞춰 자유롭게 수정하세요.

## 1. 반복 작업 정의

내가 같은 패턴으로 여러 번 하는 작업 하나를 고른다.

- 작업 이름:
	- 예: 고객 리뷰 긍정/부정 분류
- 한 번에 처리하는 항목 수 (지금 방식):
	- 예: 1개씩, 따로따로
- 한 항목당 같은 지시문을 반복하고 있는가:
	- [ ] 예 — 배치로 묶을 가치가 큼
	- [ ] 아니오

## 2. 배치 프롬프트 템플릿

지시문은 한 번, 항목은 번호로. 결과도 번호로 받는다.

```
{여기에 한 번만 쓰는 지시문}
아래 항목을 처리하고 번호 그대로 답해줘.
1. {항목}
2. {항목}
3. {항목}
...
```

내 작업에 맞춘 실제 템플릿:

```
(여기에 채우기)
```

## 3. 배치 크기 실험

같은 작업을 5개·10개·20개 묶음으로 돌려보고 정확도를 적는다.
정답은 직접 채점 — 맞은 개수 / 전체 개수.

| 배치 크기 | 맞은 개수 | 정확도 | 흐려진 항목 위치 (앞/중간/뒤) | 메모 |
| --- | --- | --- | --- | --- |
| 5개 | / 5 | % | | |
| 10개 | / 10 | % | | |
| 20개 | / 20 | % | | |

## 4. 속도·비용 메모

- 내가 쓰는 도구의 rate limit (분당 호출/토큰):
	- 예: 분당 60회 / 분당 90,000 토큰
- 막혔을 때 대응:
	- [ ] 호출 사이 간격 두기
	- [ ] 막히면 잠시 기다렸다 재시도
- 비용 절감 적용 여부
	- [ ] 캐싱 — 반복 지시문 재활용
	- [ ] 싼 모델 — 단순 작업은 작은 모델로
	- [ ] 배치 API — 급하지 않은 대량 작업
