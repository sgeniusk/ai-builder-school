# SQL 생성 + 검증 프롬프트 템플릿

매번 처음부터 쓰지 않는다. 아래 칸을 채워 붙여넣는다.

---

## 1단계 — 쿼리 생성 프롬프트

```
[스키마]
<schema-notes.md 에서 관련 테이블 복사>

[DB 환경]
- DB: MySQL 8.0 / PostgreSQL 14 / BigQuery (해당 항목 선택)
- 인덱스 있는 컬럼: <인덱스 컬럼 명시>

[요청]
<원하는 분석을 비즈니스 언어로>

[제약]
- NULL 처리를 명시해줘
- 대용량 테이블이니 풀 스캔을 피할 것
- GROUP BY 누락 없이
- 결과 컬럼명을 한국어로 별칭 붙여줘 (예: AS 일수)
```

---

## 2단계 — AI 셀프 리뷰 프롬프트

1단계 응답을 받은 뒤, 같은 채팅에서 이어서 요청한다:

```
위 쿼리에서 다음을 확인해줘:
1. 스키마와 맞지 않는 컬럼명이 있는가
2. NULL 처리가 빠진 곳이 있는가 (특히 LEFT JOIN, 집계 함수)
3. GROUP BY에 빠진 컬럼이 있는가
4. 0으로 나누기 같은 엣지 케이스가 있는가
5. WHERE 절에 인덱스 컬럼이 사용됐는가

문제가 있으면 수정된 쿼리를 다시 줘.
문제가 없으면 "이상 없음"이라고 명시해줘.
```

---

## 3단계 — 테스트 데이터 생성 프롬프트

```
위 쿼리를 검증하기 위한 테스트 데이터를 만들어줘.
- 각 세그먼트(예: 첫 구매 빠름 / 느림 / 없음)를 2행씩 포함
- INSERT INTO 구문으로
- 예상 결과도 함께 보여줘 (내가 손 계산으로 검증할 수 있게)
```

---

## 자주 쓰는 쿼리 패턴 메모

| 패턴 | 핵심 포인트 |
|------|------------|
| 첫 구매일 계산 | `MIN(orders.created_at)` + LEFT JOIN (구매 없는 유저 포함) |
| 30일 활성 사용자 | `WHERE events.created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)` |
| 코호트 분석 | `DATE_FORMAT(users.created_at, '%Y-%m')` 로 가입 월 그룹핑 |
| 누적 합계 | `SUM(amount) OVER (PARTITION BY user_id ORDER BY created_at)` |
