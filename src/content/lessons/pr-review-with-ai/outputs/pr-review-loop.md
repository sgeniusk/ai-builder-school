# PR 리뷰 루프

> 이 파일은 지음 · Stage 5 "AI에게 일감을 주다" 에서 제공되는 출발 템플릿입니다.
> AI가 짠 PR을 머지하기 전 거치는 2단계 점검 루틴입니다. 당신의 프로젝트에 맞춰 수정하세요.

## 1. 셀프 리뷰 프롬프트

코드를 짠 AI에게 그대로 붙여넣어 자기 diff를 1차 검토시킨다.

```
방금 만든 변경의 전체 diff를 다시 검토해줘. 아래 세 가지만 찾아서 보고해줘.

1. 위험한 변경 — 되돌리기 어렵거나 다른 곳에 영향을 줄 수 있는 부분
2. 요청 범위 밖 수정 — 내가 요청하지 않았는데 "그 김에" 건드린 부분
3. 빠진 테스트 — 추가했어야 하는데 빠진 테스트 케이스

각 항목을 파일·줄 번호와 함께 짧게 적어줘. 문제가 없으면 "없음"이라고만 해줘.
```

## 2. 사람 필수 점검 체크리스트

셀프 리뷰가 통과해도 아래는 사람이 직접 본다. AI 판단에 맡기지 않는다.

- [ ] **보안** — 비밀키·토큰·비밀번호가 코드나 로그에 노출되지 않는가
- [ ] **권한** — 인증·인가 로직이 의도대로인가, 누구나 호출 가능해진 엔드포인트는 없는가
- [ ] **데이터 마이그레이션** — 스키마 변경이 기존 데이터를 깨거나 잃게 하지 않는가
- [ ] **외부 호출** — 새로 추가된 외부 API·결제·이메일 호출이 정말 필요한가, 실패 시 처리가 있는가
- [ ] **되돌리기 어려운 변경** — 머지 후 롤백이 곤란한 부분(삭제·덮어쓰기)은 없는가
- [ ] **범위 일치** — 이 PR이 처음 요청한 한 가지 일만 하는가

## 4. 되먹임 규칙

리뷰에서 같은 문제가 반복되면 다음 작업 지시에 미리 넣는다.

- [ ] 이번 리뷰에서 발견한 문제가 지난 PR에서도 나왔던 것인가
- [ ] 반복된다면 → 다음 작업 브리프에 한 줄로 추가할 규칙 :
	- 예: "테스트 없이는 코드 변경 금지", "요청한 파일 외에는 수정하지 말 것"
- [ ] 프로젝트 전체에 적용할 규칙이라면 → `CLAUDE.md`에 추가할 문장 :
