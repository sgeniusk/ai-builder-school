## 계획은 있는데 매일 나아가지 못하는 이유

선언문도 썼다. 기능 목록도 있다. 근데 막상 시작하면 막막하다.

```
"오늘 뭘 해야 하지?"
→ 고민하다 30분 흘러간다.

"이 에러가 왜 나는 거지?"
→ 2시간 동안 못 풀었다.

"오늘은 좀 쉬고 내일 하면 되겠지."
→ 3일이 지났다.
```

캡스톤이 실패하는 이유는 기술이 아니라 리듬이 없어서다.

---

## 4주 빌드 루프

4주를 주별 단계로 나눈다.

```
1주차: Spec — 뭘 만들지 구체화
2주차: Build — 핵심 기능 구현
3주차: Test + Fix — 직접 써보고 수정
4주차: Ship — 배포 + 회고
```

각 주는 이 루프로 운영한다:

```
월요일: 이번 주 목표 3개 정하기
화~목: 하루 1-2 태스크 실행
금요일: 스테이징 배포 + 빌드 로그 작성
```

---

## 일일 지시 패턴

코딩 에이전트에게 매일 이 형식으로 지시한다:

```
오늘 목표: [무엇을 완성할 것인가]
맥락: [지금 상태가 어떤가, 어디까지 왔는가]
완료 기준: [이걸 보면 오늘 태스크가 끝난 것이다]
```

예시:

```
오늘 목표: 문서 업로드 후 청킹 + 임베딩 저장 완성
맥락: FastAPI 서버 기본 구조는 있음. /upload 엔드포인트만 없음.
완료 기준: PDF 파일을 업로드하면 pgvector에 임베딩이 저장되는 것
```

완료 기준이 없으면 에이전트가 "됐는데요"라고 해도 확인이 안 된다.

## 막힘 탈출 루틴

**30분 안에 풀리지 않으면** 이 순서로 탈출한다:

1. **문제를 한 문장으로 적는다**
   ```
   "pgvector에 insert는 되는데 검색 결과가 빈 배열로 나온다."
   ```

2. **에러 + 맥락을 통째로 AI에게 준다**
   ```
   이 에러가 나:
   [에러 메시지 전체]
   
   맥락:
   - pgvector 0.5.1, PostgreSQL 15
   - insert는 성공, select는 빈 배열
   - 코드:
   [관련 코드]
   ```

3. **30분 더 지나도 안 되면 그 기능을 다음 주로 미룬다**
   ```
   → 기능 X는 다음 주로. 오늘은 기능 Y로 넘어간다.
   ```

한 기능에 막혔다고 프로젝트 전체를 멈추지 않는다.

## 빌드 로그 작성

매일 4칸을 채운다. 5분이면 된다.

```
날짜: 2026-05-15

한 일:
- /upload 엔드포인트 구현 완료
- PDF 청킹 로직 작성 (500토큰/오버랩 50)

에이전트가 한 일:
- FastAPI 엔드포인트 코드 전체
- pgvector insert 쿼리

막혔던 것:
- 임베딩 차원이 맞지 않아 insert 실패 → 모델 확인 후 수정

내일:
- /query 엔드포인트 구현
- 유사도 검색 테스트
```

로그가 쌓이면 나중에 회고 쓸 때 데이터가 된다.

## 1주차 말 스테이징 공유

금요일에 "완성"을 기다리지 않고 배포한다.

```
지금 상태로 Vercel에 올려줘.
기능이 완성 안 됐어도 괜찮아.
지금 되는 것만 보여주는 스테이징이야.
```

1-2명에게 링크를 보내고 묻는다:

```
"이게 뭘 하는 건지 이해돼?
막히는 부분 있으면 바로 말해줘."
```

피드백이 없어도 괜찮다. 올렸다는 사실이 중요하다.
