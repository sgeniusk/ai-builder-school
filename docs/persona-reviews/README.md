# Persona Reviews — 10명의 시선으로 본 AI Builder School

이 폴더는 *AI Builder School의 자체 커리큘럼*을 10개 페르소나가 *각자의 입장에서* 읽고 작성한 깊은 평가 보고서를 모은다. 외부 강의 분석(`docs/dewey/`)과는 별개 — 이건 *내부 reality check*다.

## 목적

- 현재 커리큘럼이 *너무 어렵거나 뜬금없는 부분*을 페르소나 시선으로 발견한다
- 칭찬 위주의 자기 평가가 아닌 *가차 없는 솔직 평가*를 통해 우선순위를 잡는다
- 변경 제안은 *구체 파일·필드 단위*로 받아 직접 반영 가능하게 한다

## 방법론

- 각 페르소나는 **Codex CLI**가 1대1로 시뮬레이션 (`codex exec`)
- 10개를 *병렬*로 실행 (단일 세션이 아닌 진짜 동시 실행)
- 동일한 평가 양식(9 섹션)으로 보고서 작성
- 페르소나가 *직접* `phases.ts` / `lessons.ts` / 8개+ lesson MDX를 읽고 평가
- 가차 없는 솔직 평가 권장 — *비판 없는 칭찬 보고서는 무가치*로 명시

## 산출물

- **개별 보고서** — `2026-05-12-<slug>.md` × 10개 (Codex가 작성)
- **종합 합본** — `2026-05-12-synthesis.md` (모든 보고서 모인 후 메인 세션이 작성)
- **실행 로그** — `.tmp-codex-logs/<timestamp>/` (각 codex 호출의 stdout/stderr, _master.log)

## 보고서 표준 양식 (9 섹션)

1. 페르소나 자기소개 (1인칭, 1문단)
2. 커리큘럼 첫 인상
3. 깊게 읽은 lesson 목록 (왜 골랐나)
4. 너무 어려운 부분 / 막힌 지점 (구체 인용)
5. 뜬금없는 요소 / "왜 여기 있나?"
6. 빠진 부분 / 진짜 알고 싶었던 것
7. 좋은 부분 / 인상 깊은 lesson
8. lesson별 미시 피드백 (최소 8개, 표 형식)
9. 종합 의견 (verdict) + 변경 제안 3개

## 절대 하지 말 것 (페르소나에게 지시한 원칙)

- 강의 띄워주기 — *비판 없이 칭찬만* 하는 보고서는 무가치
- 페르소나 목소리를 잃고 *일반 분석가*처럼 쓰기
- lesson 본문을 안 읽고 *제목만 보고* 평가하기
- 한국어 문장을 콜론으로 끝내기
