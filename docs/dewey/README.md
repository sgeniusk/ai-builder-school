# Dewey — 외부 자료 학습 리포트

듀이는 외부 사이트·강의·아티클을 분석해 *"AI Builder School에 가르쳤으면 좋을 내용"* 을 추출하는 에이전트다. 이 폴더는 듀이의 분석 결과물을 모은다.

## 이 폴더에 들어오는 것

- 다른 강의·교육 사이트의 커리큘럼 분해 분석
- 외부 강의의 페르소나·메시징·인포그래픽 패턴 분석
- 기사·블로그 글에서 추출한 *가르침으로 전환 가능한* 콘텐츠 분석

## 이 폴더에 들어오지 않는 것

- 외부 마케팅 카피의 단순 복사
- 가격·강사 프로필·환불 정책 등 가치 판단 외 항목
- AI Builder School의 자체 implementation plan (그건 plan 파일 또는 PR에서 다룬다)
- 일회성 메모 또는 회의록

## 파일 네이밍

`YYYY-MM-DD-<source-slug>.md`

예시 — `2026-05-12-fastcampus-harness-engineering.md`

## 산출물 표준 양식

모든 듀이 리포트는 다음 구조를 따른다.

1. **Context** — 왜 이 분석을 했는가, 이 강의/사이트가 AI Builder School과 어떤 관계인가
2. **메시지 흐름 분석** — Hero → Persona → USP → 커리큘럼 노출 순서
3. **커리큘럼 전체 구조** — 시간·파트 수 포함 전체 트리 (강의면 클립 단위까지)
4. **인포그래픽/비주얼 패턴** — 사용된 다이어그램·카드·슬라이더의 구조와 메시지 운반 방식
5. **페르소나·니즈 분석** — 통증 → 응답 콘텐츠 매핑
6. **교육 방식·학습 설계 패턴** — 클립 길이, 챕터 구성, 실습 카탈로그, 사례 연구 등
7. **적재적소 반영 매트릭스 (핵심 섹션)** — 듀이 결과물은 항상 이 매트릭스로 수렴한다
	- 4 카테고리 — ① 카피·메시지 (랜딩/도입부) ② 학습 본문 (lesson MDX) ③ 정보 구조 (사이트 IA) ④ 운영 (스킬·가이드라인)
	- 각 패턴 — "어디에 (단 한 곳)" × "어떻게 변형" × "쓰지 말아야 할 곳" 세 축으로 정리
8. **차별화 권장** — 원본과 겹치지 말 것 — AI Builder School이 의도적으로 다르게 가야 할 지점
9. **우선순위** — "지금 손대도 좋은 3개" vs "별도 implementation plan 필요" 분리

## 듀이의 원칙

- **한 번에 모두 도입하지 않는다.** 패턴마다 어울리는 자리가 정해져 있다.
- **마케팅 카피와 학습 본문은 다른 자리다.** 톤·길이·반복 빈도가 다르다.
- **"어디에 쓰지 말아야 하나"를 항상 적는다.** 적재적소의 핵심은 *제한* 이다.
- **원본 강의/사이트의 가격·강사 정보는 분석하지 않는다.** 가치 판단 외 항목.
- **AI Builder School의 자체 thesis와 충돌하면 원본을 따르지 않는다.** 듀이는 흡수가 아니라 *선별* 한다.
