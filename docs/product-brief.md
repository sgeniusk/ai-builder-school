# AI Builder School — 제품 브리프

> "AI를 쓰는 사람"에서 "AI로 만드는 사람"으로.

## 1. 한 문장 요약
**AI Builder School**은 한국어권 학습자를 위한 **실전형 AI 빌더 스쿨**이다. 수동적인 강의가 아니라 Codex / Claude Code를 활용해 **직접 무언가를 만들고, 검증하고, 배포하게 하는** 학습 플랫폼이다.

## 2. 문제 정의 (Problem)
- AI 열풍은 강하지만, "뭘 공부해야 실제 일에 쓰이는가?"는 여전히 모호하다.
- 뉴스·유튜브·블로그가 너무 많아 오히려 방향을 잃는다.
- 강의를 듣고도 자기 일에는 못 붙인다 (전이 실패).
- 개발자는 Codex/Claude Code를 쓰지만, 비개발자는 진입 장벽이 높다.
- 기획자/관리자는 조직에 AI를 "어떻게 도입할지" 모르고, 크리에이터는 워크플로우를 조합하지 못한다.

## 3. 제품의 약속 (Promise)
> "한 주 단위로 실제 만든 결과물을 쌓고, AI로 일하는 사람에서 AI로 만드는 사람이 되자."

매 주마다:
- 하나의 **문제**를 이해하고,
- 최소한의 **개념**을 배우고,
- Codex / Claude Code에게 **미션**을 내리고,
- 작은 것을 **빌드**하고,
- **검증 체크리스트**로 확인하고,
- 결과물을 **배포 또는 공유**하고,
- "AI가 한 것"과 "사람이 판단한 것"을 **회고**한다.

## 4. 대상 학습자 (Target Learners)
| 페르소나 | 핵심 니즈 | AI Builder School이 주는 것 |
|---|---|---|
| 일반 직장인(office) | 업무 생산성 향상 | AI Work OS 플레이북 |
| 기획자·관리자(planner) | 조직에 AI 도입 | 도입 전략 + 검증 체크리스트 |
| 개발자·앱 빌더(developer) | Codex/Claude Code로 빠른 개발 | 코딩 에이전트 루틴 + RAG/Agent 템플릿 |
| 콘텐츠 크리에이터(creator) | 글/영상 자동화 | 멀티모달 파이프라인 |
| AI 엔지니어 심화(engineer) | RAG/Agents/Evals 심화 | 실습 중심 커리큘럼 |

## 5. 핵심 차별점 (Why this, not that)
- **도구 중립**: Codex, Claude Code, Cursor Agent 무엇을 써도 동작.
- **한국어 중심**: UI/용어/예시가 한국 직장과 조직 맥락.
- **빌드 중심**: 매 레슨이 **산출물(Deliverable)** 로 끝남.
- **검증 중심**: 환각·프롬프트 인젝션·과의존 등을 학습자가 직접 확인.
- **비개발자 친화**: `src/content/*.ts`에서 문서처럼 레슨을 추가할 수 있다.

## 6. 학습 루프 (7-step Loop)
모든 레슨에 고정된 7단계가 반복된다. 이 리듬 자체가 제품의 형태다.

1. 문제 이해 (Problem)
2. 최소 개념 (Concept)
3. Codex / Claude 미션 (Mission)
4. 빌드 (Build)
5. 검증 (Verify)
6. 배포·공유 (Ship)
7. 회고 (Reflect)

## 7. 정보 설계 (IA Summary)
9개 핵심 경로: `/`, `/start`, `/curriculum`, `/curriculum/[phaseSlug]`, `/lessons/[lessonSlug]`, `/tracks`, `/projects`, `/templates`, `/about`.

## 8. MVP 경계
In-scope: 정적 사이트, 12 Phase, 32+ 레슨, 5 트랙, 프로젝트/템플릿 카탈로그, 에이전트 지침.
Out-of-scope: 계정, 결제, DB, 커뮤니티, 실시간 샌드박스, 영상.

## 9. 성공 지표 (Phase 1)
- 학습자가 `/start`에서 트랙을 선택 → 첫 레슨까지 3클릭 이내.
- 한 주차 레슨을 마친 학습자가 **산출물**을 보유.
- 비개발자 기여자가 레슨 1개를 스스로 추가 가능.
- 검색 엔진에서 한국어 "AI 빌더 스쿨" 관련 키워드로 검색 가능.

## 10. 브랜드 톤
- 실전적 (Practical)
- 빌더 중심 (Builder-oriented)
- 과장 금지 (No hype)
- 존중 (학습자의 시간을 아낀다)
